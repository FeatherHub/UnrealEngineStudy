**“풀 수 있는 문제와, 현실적으로는 포기해야 하는 문제의 경계”**

algorithm이 도시의 교통 규칙이라면
data structure는 도시의 도로망이에요.

arrays, stacks, queues, linked lists
hash tables
binary search trees
red-black trees

같은 연산도 data structure에 따라 cost가 달라져요.

연산을 빠르게 만들고 싶다면, 먼저 데이터가 사는 형태를 잘 설계해야 한다.

hash table은 “주소를 바로 찾는 감각”
BST는 “정렬된 질서를 유지하면서 탐색하는 감각”
red-black tree는 “질서를 유지하되 편향을 막는 자가교정 감각”

Dynamic Programming
overlapping subproblems + optimal substructure

미로를 매번 처음부터 탐색하는 대신,
한 번 계산한 갈림길의 결과를 메모해 두는 방식이에요

기억을 사용하는 계산

Greedy

매 순간 가장 좋아 보이는 선택을 해도 전체적으로 최적이 되는 구조가 있어요.
사실은 문제 구조가 매우 특별할 때만 통하는 방식이에요.
증명되어야 할 전략으로 다뤄요.

Amortized Analysis
어떤 연산은 한 번은 비싸지만, 긴 흐름 전체로 보면 싸요.
가끔 resize가 비싸더라도, 전체 push들의 평균 cost는 작을 수 있어요.
시간의 흐름 전체에서의 경제성을 보는 눈을 길러줘요.

Augmenting

기존 structure에 메타데이터를 얹어서 더 강한 질의를 가능하게 해요.

B-Trees
이건 main memory가 아니라 
disk / external memory 현실을 강하게 의식하는 구조예요.
CPU 비교 횟수만이 아니라, 실제 저장장치 접근 패턴까지 고려해요.

Disjoint Sets

서로소 집합 구조는 “누가 누구와 같은 그룹인가”를 매우 빠르게 관리해요.

Graph Algorithms
세상을 점과 선의 관계망으로 보는 감각을 배워요.

현실의 많은 시스템이 graph예요.
도로망
네트워크
dependency

state transition

graph algorithm은 일종의 “관계의 물리학” 같아요

BFS / DFS

탐색의 기본 호흡이에요.
graph를 걷는 방식 
BFS는 파동처럼 퍼져나가며 가장 가까운 층을 먼저 봐요.
DFS는 굴을 깊게 파고 들어가며 구조를 드러내요.

MST
모든 지점을 연결하되 총 비용을 최소화하는 문제예요.

전선 깔기, 도로 연결, 클러스터 구조 이해 같은 데 이어져요.

Shortest Paths
길찾기

Flow / Matching

Parallel Algorithms

Insertion sort

“아직 전체는 정렬되지 않았지만,
앞부분은 항상 정렬된 상태로 유지한다”

한 번에 전체를 해결하지 않아요.
매 단계마다 정렬된 작은 세계를 조금씩 키워 나가요.

이미 정렬된 부분과 아직 정렬되지 않은 부분을 구분하는 감각
배열 전체를 한 덩어리로 보지 않고,
왼쪽: 이미 정리된 영역
오른쪽: 아직 처리되지 않은 영역

“질서가 확보된 부분”을 점점 넓혀 가는 방식으로 움직여요.

loop invariant
algorithm은 상태를 유지하면서 전진한다는 감각
insertion sort는 매 단계마다 어떤 성질을 유지해요.
반복이 시작될 때마다 앞부분은 정렬되어 있다

반복문이 한 번 돌기 전에도 참이고,
한 번 돌고 나서도 계속 참이고,
마지막에 반복문이 끝났을 때는 그 성질이 곧 정답이 되게 만드는
반복문의 중심 약속

Loop invariant는 반복문의 correctness를 붙잡는 손잡이
반복 과정 전체를 통제하는 중심 약속이에요

Divide-and-conquer
작은 문제들이 원래 문제와 닮아 있어야 한다

두 개의 이미 정렬된 리스트가 있으면
그 둘을 하나로 합치는 것은 쉬워요.

각 리스트의 맨 앞만 비교하면 되기 때문이에요

카드 더미 두 개에서
맨 위 카드 둘만 보고
더 작은 것을 출력 더미로 보내는 느낌

이미 정렬된 두 흐름

insertion sort는 많은 원소가 서로 계속 부딪혀요
merge sort는 먼저 분할해서 복잡도를 낮춘 뒤 질서 있게 합쳐요

insertion sort
원소를 하나 넣을 때마다
기존 질서를 밀어내며 자리를 찾아야 한다
그리고 나머지 부분이 무질서(chaos)이다

문제의 구조 자체를 재편성해서 푸는 방법

incremental 방법과 divide-and-conquer 방법

하나는 질서를 조금씩 확장하고,
다른 하나는 문제를 분할하고 합성해요.

먼저, 책상 위 서류 더미를 손으로 한 장씩 정리하는 방법을 보여줘요.
그 과정에서 “정리된 부분”을 유지하는 원리를 배우게 해요.
그다음, 정리하는 데 시간이 얼마나 드는지를 세어 보게 해요.
마지막으로는 아예 방식을 바꿔서,
서류 더미를 둘로 나누고 각각 정리한 뒤 다시 합치는 전략을 보여줘요.

줌아웃 기술
정확한 초 단위 숫자에 너무 집착하지 말자
constant factor와 lower-order term에 너무 매달리지 말자
input size가 커질 때 무엇이 본질적으로 지배적인지만 보자

4n^2 + 100n + 500
input이 충분히 커지면 결국 n^2 항이 게임을 지배

O-notation
asymptotic upper bound
충분히 큰 n부터는
이 함수가 대략 g(n)보다 더 빨리 자라지 않는다

Ω-notation
asymptotic lower bound
충분히 큰 n부터는
이 함수가 적어도 g(n)만큼은 자란다

O가 천장이라면, Ω는 바닥이에요.
함수가 아무리 흔들려도 그 바닥 아래로는 내려가지 않아요.

어떤 algorithm이 최소한 이 정도 시간은 필요하다
이보다 더 빠를 수는 없다

Θ-notation
tight bound
충분히 큰 n부터는
이 함수가 위로도 g(n)급이고, 아래로도 g(n)급이다

3n log n과 1000n log n은
디테일은 다르지만 asymptotically는 같은 family예요.
이건 엄청난 추상화예요.
running time의 창문과 벽돌을 지우고
골격만 보게 만드는 장이에요

polynomial은 algorithm analysis의 기본 지형이에요.
