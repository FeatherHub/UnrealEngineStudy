  4-3 결과를 FNiagaraDynamicDataBase 으로 패키지

Render thread 로 enqueue

9
CPU→GPU 업로드
VertexFactory 가 셰이더에 vertex 당 / instance 당 입자 attribute 를 공급 해야 한다
셰이더는 GPU 메모리에서만 읽을 수 있으므로, DynamicData 의 CPU 바이트들이 GPU 로 업로드되어야 한다.

이 업로드가 매 frame 일어난다  (CPU 시뮬은 매 tick 새 데이터를 만들어 내므로)
GPU 모드에서는 데이터가 처음부터 GPU 버퍼에 살기 때문에 업로드 자체가 필요 없다

FPrimitiveSceneProxy가 GetDynamicMeshElements와 DrawStaticElements 콜백을 통해 FMeshBatch를 renderer에 제출한다
dynamic path는 FMeshBatch를 매 프레임 다시 만든다

DrawStaticElements
  = SceneProxy 생성 시점 등에 static mesh element를 등록/캐시

GetDynamicMeshElements
  = 렌더링할 때마다 현재 View 기준으로 dynamic mesh element를 모음

이름이 완벽히 대칭이 아니죠.

Static:
  "한 번 등록해둘 테니 PDI에 그려 넣어라."
  → DrawStaticElements(FStaticPrimitiveDrawInterface* PDI)

Dynamic:
  "이번 View에서 필요한 mesh batch들을 collector에 넣어라."
  → GetDynamicMeshElements(..., FMeshElementCollector& Collector)
