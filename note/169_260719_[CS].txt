상품: happens-before 쌍 a→b 하나. 
통화: 하드웨어 동전(fence 명령, cache line의 소유권 이전). 

가격표는 아키텍처마다 다르다

x86은 기본 요금에 상당한 순서가 포함된 패키지 상품이고(TSO: 웬만한 acquire/release가 공짜), 
ARM은 순서를 낱개로 파는 시장이라 fence마다 지불한다. 

경쟁이 붙은 CAS는 cache line이 core 사이를 핑퐁하며 가격이 수요와 함께 오른다
혼잡 요금제

묶음 할인도 있다: acquire 하나를 사면 transitivity 덕에 release 이전의 모든 쓰기가 딸려 온다.

side channel

추상은 "이 차이들은 은닉된다"고 약속하지만, 물리는 계속 상관을 만든다

비밀번호 비교가 틀린 글자에서 일찍 return하면 그 시간 차가 비밀을 방송하고, 
투기 실행이 남긴 cache 흔적은 "투기는 보이지 않는다"는 추상의 약속을 뚫는다

"차이를 만드는 차이"의 무허가판
은닉했다고 선언한 차이가 기어이 차이를 만드는 것.

refresh와 ECC라는 손이 "가만히 있음"을 상연한다.

상태를 잡음에 맞서 유지하는 우물 자체가 구조물이라 열역학 제2법칙 아래에서 삭아 간다

---

단순한 기계(in-order, buffer 없음)는 전체 순서를 공짜로 준다
속도를 사려고 우리가 순서를 팔았다

out-of-order 실행, 다중 core

memory model이란 그렇게 팔아치운 순서를 부분적으로 되사는 거래소다

---

cache line(64 byte)의 배타적 소유를 요구하므로, 
여러 core가 한 counter를 CAS하면 소유권이 core 사이를 왕복한다
이동 한 번이 interconnect를 건너는 수십~수백 cycle이고
수요가 몰릴수록 한 번의 갱신 단가가 오른다

false sharing
서로 무관한 변수 둘이 우연히 같은 line에 살면 남의 혼잡 요금을 대신 낸다
자료구조들이 cache line 크기로 칸을 띄우는 이유 (padding)

---

reorder buffer(ROB)

모든 명령은 진입 시점에 프로그램 순서대로 장부에 한 칸씩 받고, 
실행은 난장판으로 하되, 퇴장(retire)은 장부 머리부터만 순서대로 한다

퇴장한 역사가 외관이고, 실행된 난장판이 실재
파기된 비공식 역사가 cache에 남긴 흔적은 지워지지 않는다

---

약속은 "모든 주소는 균등하다",
실재는 L1 ~4 cycle, L2 ~12, L3 ~40, RAM ~수백 cycle의 가파른 지형

---

DRAM capacitor에 담긴 것은 수만 개 전자 규모의 전하이고
누설 전류가 이를 상시 빼돌린다

수십 ms가 지나면 판독 불가능해질 수 있으므로
memory controller는 모든 행을 약 64 ms 주기로 읽고-되쓰기 한다

게다가 읽기 자체가 파괴적이라(그릇을 열면 비워진다) 
sense amplifier가 판정 직후 복원해 준다

DRAM의 bit는 "저장되어 있는" 것이 아니라 
초당 십수 회 재발화되고 있다

가만히 있음의 실체는 반복해서 다시 말해짐이다
상태의 지속이 사실은 과정의 지속이었던 것

---

refresh로도 못 막는 것

64 bit를 저장할 때 72 bit를 쓴다. 

추가 8 bit는 겹치는 부분집합들의 parity인데
어느 한 bit가 뒤집히면 어긋난 parity들의 패턴(syndrome)이 뒤집힌 자리의 이름을 댄다
읽으면서 고친다(1개 정정, 2개 검출)
