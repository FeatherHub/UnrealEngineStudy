tCL, tRCD

t는 시간(Time) 혹은 클럭 사이클(Clock Cycles)

Column Address Strobe Latency
메모리 컨트롤러가 RAM에 특정 데이터를 요청한 후, RAM이 해당 데이터를 준비하여 출력을 시작할 때까지 걸리는 시간


RAS to CAS Delay
데이터가 들어있는 행(Row)을 활성화한 후, 그 행 안에서 정확한 열(Column) 주소에 접근할 수 있을 때까지 기다려야 하는 대기 시간
도서관에서 원하는 책이 있는 '층(Row)'을 찾아 올라간 뒤, 그 층의 구체적인 '구역(Column)'으로 이동하기 전까지 필요한 시간

RAM 은 USB 로 가는 데이터 경로의 "경유지"가 아니라 staging 장소

(1) CPU 가 RAM 에 데이터를 씀 — core → on-die interconnect → IMC → DDR bus → DIMM.

(2) CPU 가 USB host controller 의 레지스터에 "이 RAM 주소의 데이터를 보내라"고 MMIO 로 명령함.

(3) USB host controller 가 bus master 로서 직접 RAM 을 DMA 로 읽음 — controller → PCIe → IMC → RAM, 다시 반대 방향으로 데이터.

(4) USB host controller 가 USB 프로토콜대로 패킷화해서 USB 버스로 송출.

(5) USB flash drive 내부 컨트롤러가 NAND 에 씀.

DDR bus

Source-synchronous
전송 거리가 짧고 신호 품질이 좋아야만 성립하는 스타일


Burst-oriented
한 번의 READ/WRITE 명령이 연속된 8~16 개의 beat 를 뱉어냄
주소 하나로 연속 데이터를 퍼내는 식
DRAM 의 row buffer 구조를 활용하기 위함


Master-slave 완전 비대칭
memory controller 만이 명령을 발행하고, DRAM 은 수동적으로 응답하기만
DRAM 은 자발적 행위 능력이 없고, 거대한 수동 저장소일 뿐


Parallel
데이터선이 병렬로 다수
DDR4 채널은 64 data + 8 ECC
DDR5 는 32+32 두 sub-channel


프로토콜 오버헤드가 거의 없고, 
DRAM 의 물리적 특성(row activate, precharge, refresh)에 맞춰 타이밍이 얇게 깎여 있어.

On-die interconnect
packet-switched network on chip


Intel 의 Ring bus
코어, LLC slice, IMC agent, System Agent 등이 stop 으로 배치된 링
각 stop 에 agent 가 있고 packet 을 주고받는다


LLC
Last Level Cache, 보통 L3 캐시
각 코어 옆에 Slice 단위로 쪼개져 배치
모든 코어가 이 슬라이스들에 분산된 전체 L3 캐시 용량에 접근


System Agent
과거 Northbridge의 기능을 CPU 내부로 통합한 제어부
이 안에는 IMC(Integrated Memory Controller, 통합 메모리 컨트롤러), 디스플레이 엔진(iGPU 출력), PCIe 컨트롤러, 전력 제어 유닛(PCU)
등이 포함


Skylake 이후 server 급은 Mesh
2D grid 의 각 tile 에 agent, packet 이 mesh routing 됨


이 네트워크가 cache coherence protocol (MESI 계열) 을 first-class 로 지원
"읽기/쓰기 요청"만이 아니라 "누가 이 캐시라인을 소유 중인가"의 상태 메시지까지 포함
DDR bus 와는 의미론적 층위가 다르다


DDR 은 단순한 memory 요청 운반
on-die fabric 은 coherent distributed system 의 메시지 버스

Bus master

고전 PC 에서는 CPU 만이 master 였고, 모든 I/O 는 CPU 가 byte 단위로 중개 (PIO, programmed I/O)

DMA(Direct Memory Access) 
장치에게 bus 를 쓸 권한을 임시로 넘기는 것
이게 더 급진화돼서 — 모든 PCIe endpoint 가 잠재적 bus master

GPU, NVMe, NIC, USB host controller 전부가 CPU 없이 직접 RAM 을 읽고 쓸 능력을 가짐

---

physical address space 는 RAM 이 아니다

physical address space 는 "routing table 의 key space"다.

CPU 는 mov [0x...], 0x42 같은 store 를 실행해. 
CPU 는 이 주소가 RAM 인지 device register 인지 알지 못해
CPU 는 그냥 "이 주소에 이 값을 써라"는 store 를 on-die interconnect 에 던져.

interconnect 상에 address decoder 가 있어
부팅 시 firmware(UEFI/BIOS) 가 설정한 mapping 에 따라

주소가 RAM 영역 → IMC 로 라우팅 → DDR 트랜잭션
주소가 PCIe MMIO 영역 → IIO(Root Complex) 로 라우팅
주소가 local APIC 영역 → APIC 로 라우팅 (Advanced Programmable Interrupt Controller)

---

APIC
과거의 PIC를 대체
다중 코어 환경에서 인터럽트 신호를 배분하기 위해 설계

Local APIC (LAPIC)
각 CPU 코어 내부에 하나씩 존재
해당 코어에 직접 전달되는 인터럽트를 관리
주소가 Local APIC 영역으로 지정된 패킷은 외부로 나가지 않고, 링 버스에서 각 코어의 LAPIC 정류장으로 라우팅

I/O APIC
보통 칩셋(PCH)이나 System Agent 근처의 I/O 서브시스템에 위치
외부 장치(키보드, 마우스, 네트워크 카드 등)에서 발생하는 인터럽트 신호를 받아 어떤 코어(LAPIC)로 보낼지 결정

---

MMIO는 별도의 명령 집합이 아니야
평범한 load/store 인데, 목적 주소가 우연히 device 에게 매핑돼 있을 뿐

CPU의 ISA에 "장치와 말하는 명령" 같은 걸 추가할 필요가 없고, 주소 공간이라는 기존 추상을 재사용해서 I/O 까지 포섭

---

부팅 시 어떻게 routing 이 설정되는가

PCIe device 는 각자 BAR (Base Address Register) 를 갖고 "나는 이 만큼의 MMIO 공간이 필요" 라고 선언

UEFI 가 enumeration 중에 이 요청을 모아서 physical address space 안의 각 BAR 에 non-overlapping range 를 배정 
그 range 를 해당 device 로 라우팅하도록 decoder 를 프로그래밍

이 과정이 끝나면 address space 가 "RAM | device A 영역 | device B 영역 | ..." 로 분할된 map 이 돼

---

UEFI (Unified Extensible Firmware Interface)

과거 **BIOS(Basic Input/Output System)**를 대체

하드웨어 초기화: 전원을 켜자마자 CPU, 메모리, 저장장치 등을 점검하고 사용할 준비를 마칩니다
주소 할당 (Resource Allocation): PCIe 장치의 **BAR(Base Address Register)**를 읽어 MMIO 주소 범위를 배정
부트 로더 실행: 하드웨어가 준비되면 저장장치에 있는 OS 부트 로더(예: Windows Boot Manager)를 찾아 제어권을 넘겨줍니다

BIOS와의 차이
대용량 지원: 2.2TB 이상의 고용량 디스크(GPT 파티션)를 인식
보안 부트 (Secure Boot): 신뢰할 수 없는 악성 소프트웨어가 부팅 과정에 개입하는 것을 차단
속도: 16비트 모드로 작동하던 BIOS와 달리 32비트 또는 64비트 모드로 작동
유연성: 마우스 입력이 가능한 그래픽 인터페이스를 제공

---

GPT (GUID Partition Table)

물리적인 하드 디스크나 SSD에서 파티션(영역)을 나누고 관리하는 표준 규격
과거 MBR(Master Boot Record) 대체 
UEFI 표준의 일부로 도입

MBR은 주소 지정 방식의 한계로 최대 2.2TB까지만 인식
GPT는 **9.4ZB(제타바이트)**까지 지원 (1ZB = 약 10억 TB)

MBR은 기본 파티션을 4개까지만 만들 수 있으나
GPT는 128개 이상의 파티션을 생성할 수 있습니다

파티션 정보(헤더)를 디스크의 맨 앞뿐만 아니라 맨 뒤에도 복사본으로 저장합니다. 
앞부분이 손상되어도 뒷부분의 데이터를 통해 복구가 가능합니다.

파티션 테이블의 오류를 감지하기 위해 순환 중복 검사(CRC)를 수행하며, 
데이터 오염 시 이를 스스로 감지할 수 있습니다.

device 는 자기 레지스터들에 write 가 오길 기다리고 있다가, write 가 오면 그 write 자체를 "명령"으로 해석

USB host controller 의 "DoorBell" 레지스터에 1 을 쓰면 "새 descriptor 가 있으니 확인해라"라는 뜻
CPU 의 명령은 *"장치야 이걸 해라"*가 아니라 "이 주소에 이 값을 써라" 로 표현

컴퓨터가 처리할 수 있는 디지털 데이터로 변환해주는 장치입니다.

외부 영상 신호 → 캡처카드 프로세서 → PCIe 버스 → PCH → DMI 링크 → CPU/RAM


DMI (Direct Media Interface)

CPU와 PCH 사이의 모든 통신은 DMI 링크를 통해 이루어집니다.

PCH에 연결된 모든 주변기기의 데이터는 DMI를 거쳐야만 CPU와 메모리에 도달할 수 있습니다
CPU가 PCH에 연결된 장치들에 명령을 내리거나 상태를 확인할 때도 이 통로를 사용합니다.

  피연산자로부터 virtual address 계산

  TLB miss 면 page table walk

L1 → L2 캐시

System Address Decoder (SAD)
  System Agent 안에 있음
  흔히 "the address decoder" 라 부르는 게 이것
  이 주소가 DRAM 영역인가, MMIO 영역인가
  DRAM 이면 어느 채널인가
  MMIO 면 어느 root port / DMI / UPI 로 보낼 것인가


DRAM 이면

IMC 내부의 Target Address Decoder (TAD) 로 "어느 채널, 어느 rank, 어느 bank, 어느 row/column" 까지 계산 → DDR 트랜잭션

MMIO 이면

SAD 판정에 따라 IIO 안의 해당 Root Port / DMI / UPI 로 라우팅 
그 Root Port 안의 BAR 매칭 로직 이 어떤 endpoint 의 어느 BAR 영역인지 식별
TLP 로 포장해서 링크로 내보냄.

SPD (Serial Presence Detect)

모든 RAM 모듈에는 아주 작은 크기의 EEPROM(비휘발성 메모리) 칩이 붙어 있는데, 
여기에 해당 RAM의 상세 정보가 기록되어 있습니다.

"UEFI/BIOS"는 부팅 시 "SMBus"를 통해 이 칩에 접속하여 다음 정보를 읽어옵니다.

동작 속도 및 타이밍: "이 RAM은 3200MHz에서 안정적으로 돌아가고, CL 값은 22구나."
작동 전압: "1.2V를 공급해주면 되겠네."
용량: "이 메모리는 16GB구나."


사용자가 일일이 RAM 설정을 입력하지 않아도, CPU의 "IMC"가 "SPD" 정보를 보고 메모리를 최적의 상태로 가동합니다

MSI-X
