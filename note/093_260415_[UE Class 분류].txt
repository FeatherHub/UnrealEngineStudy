2. 에셋 층

UMaterial

셰이딩 모델(Lit, Unlit, Subsurface 등), 블렌드 모드, 양면 여부 같은 구조적 선택과 노드 그래프 전체를 포함
"마스터" 에셋


UMaterialInstance

UMaterial의 파라미터만 오버라이드하는 경량 에셋
노드 그래프를 소유하지 않고, 부모 머티리얼이 노출한 파라미터의 값만 저장한다
셰이더 코드는 공유하고 파라미터만 달리하는 것

UStaticMesh(공유)와 UStaticMeshComponent(인스턴스별 오버라이드)의 분리와 동일한 압력에서 태어났지만, 
에셋 층 안에서 이미 발생했다.


UMaterialInstanceDynamic(MID)
런타임에 코드로 생성되는 파라미터 오버라이드
UMaterialInstance가 에디터에서 사전 제작되는 에셋이라면, 
MID는 "실행 중에 머티리얼 색상을 빨간색으로 바꾸겠다"는 런타임 행위

패키지에 직렬화되지 않고, 요청 시 컴포넌트 레벨에서 생성되어 GC가 회수할 때까지 산다


그래프 변경(셰이더 재컴파일, 초 단위)
파라미터 프리셋 변경(에셋 교체, 밀리초)
파라미터 값 변경(유니폼 업로드, 마이크로초)

---

3. 컴파일된 셰이더 층 

Mesh에서 FStaticMeshRenderData가 "GPU가 소비할 수 있는 형태로 정리된 기하 데이터"
Material에서는 **FMaterialShaderMap이 "GPU가 실행할 수 있는 형태로 컴파일된 셰이더 프로그램"**


FMaterial
UMaterialInterface가 GetRenderProxy()를 통해 FMaterialRenderProxy를 반환
FMaterialRenderProxy가 FMaterial을 참조


FMaterialResource
셰이더 permutation 공간의 한 점에 해당하는 컴파일 결과물
UMaterial 하나가 수십 개의 FMaterialResource를 소유할 수 있다


FMaterialShaderMap
머티리얼이 사용되는 모든 패스(베이스 패스, 뎁스 패스, 섀도우 패스, 벨로시티 패스 등)와 
모든 버텍스 팩토리(스태틱 메시, 스켈레탈 메시, 스플라인 메시, 랜드스케이프 등)의 조합 집합

하나의 머티리얼이 베이스 패스에서 StaticMesh와 SkeletalMesh에 모두 쓰일 수 있으므로, 각 조합마다 별도 셰이더가 필요하다. 
이 조합 폭발이 UE의 셰이더 컴파일이 느린 근본 원인이다


FShader
개별 셰이더 프로그램(버텍스 셰이더, 픽셀 셰이더 등)의 래퍼
아래에 FRHIShader가 있고, 이것이 플랫폼별 네이티브 셰이더 객체(ID3D12PipelineState, VkShaderModule)에 매핑


기하 데이터는 "컴파일"되지 않는다. 버텍스 버퍼는 포맷 변환과 최적화를 거치지만, 프로그램으로 변환되지는 않는다. 
Material의 셰이더 층은 "고수준 기술(description)에서 기계어로의 변환"이라는 컴파일러 단계가 추가된다
이 변환이 플랫폼 × 퀄리티 × 패스 × 버텍스팩토리의 곱집합 차원에서 수행된다

4 컴포넌트 층의 부재

Material은 자신의 전용 컴포넌트를 갖지 않고 (e.g. UMaterialComponent)

머티리얼은 독립적으로 월드에 존재하지 않고, 반드시 어떤 표면 위에 존재한다

개별 소리들이 USoundSubmix를 통해 계층적으로 라우팅되고, 
각 서브믹스에서 이펙트 체인(리버브, EQ, 컴프레서 등)이 적용된 후, 
최종적으로 마스터 서브믹스에서 합산되어 출력된다

소리는 합산 가능(additive)하지만 시각은 그렇지 않기 때문이다. 
100개의 소리는 하나의 출력으로 믹스되어야 하지만, 
100개의 메시는 각각 독립적으로 래스터라이즈된다 (예외: 반투명)

Model-View-ViewModel 플러그인
UI 아키텍처 패턴
UMG 위젯(View)과 게임 데이터(Model) 사이에 ViewModel 층을 두어 양방향 바인딩을 제공


Mass Entity Framework
Entity Component System 패턴의 UE 구현
UMassProcessor가 시스템(로직), FMassFragment가 컴포넌트(데이터)


Optimus
컴퓨트 셰이더 기반 변형(deformer) 그래프 시스템
Material Editor가 "표면 외관의 그래프 편집"이라면, Optimus는 "표면 형태의 그래프 편집"


DMX — Digital Multiplex
무대 조명 제어의 산업 표준 프로토콜
UE에서 물리적 조명 기구를 실시간 제어한다.


MVR 
조명 디자인 교환 포맷(GDTF/MVR)
조명 디자이너의 가상 무대 설계를 UE에 임포트


OSC — Open Sound Control
네트워크를 통해 실시간 파라미터를 교환하는 프로토콜
라이브 퍼포먼스, 인터랙티브 설치 등에서 사용


Interchange 
UE5의 차세대 에셋 임포트/익스포트 프레임워크
FBX/USD/GLTF 등을 통합된 파이프라인으로 처리
기존 UFactory 기반 임포트를 대체


LiveLink 
실시간 스트리밍 데이터를 UE에 연결하는 프레임워크
모캡, 페이셜 캡처, 가상 카메라 등
