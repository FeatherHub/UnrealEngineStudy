futex

비어 있으면 걸쇠만 딸깍(관리인 안 부름),

optimistic concurrency
낙관적으로 일하고, 커밋하는 순간에만 충돌을 확인한다

문서가 지금 버전 5다. 그걸 열어(읽어) 로컬에서 고친다.
다 고치고 "저장"을 누르는 순간 시스템이 확인한다 — "이 문서 아직 버전 5?"

그대로 5면 → 수정이 반영되고 버전 6이 된다. (성공)
그새 누가 저장해서 6이 됐으면 → "편집 충돌, 누가 먼저 바꿨어요. 다시 하세요." (실패)

여기서 아무도 문서를 잠그지 않았다. 
다들 자유롭게 편집했고, 충돌은 저장(커밋) 순간에만 잡아 되돌린다.

return false = 그새 세상이 변했다(남이 끼어듦) → 내 결과는 낡은 전제 위에 있으니 버리고 다시

Pimpl idiom

public 객체는 "손잡이(handle)" — 얇고, 값처럼 들고 다니고 복사·이동하는 것
숨은 객체는 "몸통(body)"

compilation firewall
body 내부를 바꿔도 그 변경이 #include 그래프를 타고 재컴파일로 번지지 않고 .cpp 안에 갇힌다
Pimpl은 그 의존을 forward declaration로 끊는다

Pimpl의 비용

inlining 무력화

구현이 .cpp(opaque) 뒤라 컴파일러가 header 경계 너머로 accessor를 inline 못 한다. 
사소한 getter가 진짜 함수 호출이 된다

getter가 cpp에 정의돼 있으면 그 TU 안에서만 body가 보이고
호출자는 header의 선언만 봐서 
TU 경계를 넘는 실제 함수 호출을 emit할 수밖에 없다.

---

LTO(link-time optimization)

linker가 TU 너머로 inline할 수 있어 getter body는 inline된다

Self

FThread 생성 → OS thread 가 ThreadFunction() 실행 시작
사용자가 Detach() 호출 → FThread::Impl.Reset() 로 handle 참조 즉시 해제
만약 Impl 이 유일한 참조였다면 → FThreadImpl 이 지금 삭제됨

FThread::Impl = "주인이 신경 쓰는 동안(handle을 쥔 동안)".
Self = "thread가 실제로 도는 동안".

주인이 먼저 놓을 수도(detach), 
thread가 먼저 끝날 수도(그럼 Self가 먼저 풀리고 handle이 Join까지 붙잡음)

refcount(TSharedPtr)가 "마지막 owner가 떠날 때 삭제"를 자동 처리

UE::Tasks

models computation as a dynamically built DAG

---

"completed" means the body has executed and all nested tasks have completed, 
which is stronger than "the function returned"

Edges are prerequisite → subsequent relations
completion of a node unlocks its subsequents

---

thin Pimpl handle (TRefCountPtr)
over a heap-allocated shared state, Private::FTaskBase, 
which embeds a LowLevelTasks::FTask, the unit the scheduler actually runs

Handles (FTask, TTask<T>, FTaskEvent) are cheap to copy and only add typed accessors

---

Launch(DebugName, Body, [Prerequisites], Priority, ExtendedPriority, Flags) → TTask<TInvokeResult_T<Body>>
the single entry point. 

FTask is the untyped handle; 
TTask<T> adds GetResult() (blocks until completion, returns T&). 

FTaskEvent is a body-less graph node you Trigger() manually — the recommended replacement for FEvent
FPipe serializes its tasks (a lightweight substitute for a named thread)

Combinators: Wait(Tasks), WaitAny(Tasks), Any(Tasks), plus AddNested, MakeCompletedTask, FCancellationToken, FTaskConcurrencyLimiter.

---

// fire-and-forget
Launch(UE_SOURCE_LOCATION, [] { DoWork(); });

---

// result retrieval
TTask<int> Sum = Launch(UE_SOURCE_LOCATION, [] { return 2 + 3; });
int Value = Sum.GetResult();                     

---

// dependencies: C runs after A and B complete
FTask        A = Launch(UE_SOURCE_LOCATION, [] { Prepare(); });
TTask<FData> B = Launch(UE_SOURCE_LOCATION, [] { return LoadData(); });
FTask        C = Launch(UE_SOURCE_LOCATION,
                        [B]() mutable { Consume(B.GetResult()); },   // completed by now
                        Prerequisites(A, B));

---

// manual gate in the graph
FTaskEvent Ready{ UE_SOURCE_LOCATION };
FTask D = Launch(UE_SOURCE_LOCATION, [] { AfterReady(); }, Prerequisites(Ready));
Ready.Trigger();

---

// nested task: parent not completed until child is, but no worker blocks
Launch(UE_SOURCE_LOCATION, []
{
    AddNested(Launch(UE_SOURCE_LOCATION, [] { SubStep(); }));
});

---

// pipe: tasks never run concurrently → lock-free access to shared state
FPipe Pipe{ TEXT("StreamerPipe") };
Pipe.Launch(UE_SOURCE_LOCATION, [this] { Cache.Add(Item); });
Pipe.Launch(UE_SOURCE_LOCATION, [this] { Cache.Flush(); });  // strictly after Add

---

// background priority, cooperative cancellation
FCancellationToken Token;
Launch(UE_SOURCE_LOCATION,
       [&Token] { while (!Token.IsCanceled()) Step(); },
       ETaskPriority::BackgroundNormal);
Token.Cancel();

---

// bounded parallelism: ≤ 4 concurrent, Slot ∈ [0..4) indexes per-lane buffers
FTaskConcurrencyLimiter Limiter(4);
for (FChunk& Chunk : Chunks)
    Limiter.Push(UE_SOURCE_LOCATION, [&Chunk](uint32 Slot) { Process(Chunk, Scratch[Slot]); });
Limiter.Wait();

---

// combinators
Wait(TArray<FTask>{ A, C, D });        // all completed
int32 First = WaitAny(SomeTasks);      // index of first completed
FTask AnyDone = Any(SomeTasks);        // node completing when any completes
