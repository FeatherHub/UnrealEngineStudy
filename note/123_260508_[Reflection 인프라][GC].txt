struct FUObjectItem
{
    UObjectBase* Object;        // pointer to the UObject
    int32 Flags;                // EInternalObjectFlags
    int32 ClusterRootIndex;     // for GC clustering
    int32 SerialNumber;         // for weak pointer validation

GC가 오브젝트의 내부를 볼 필요 없이 엔트리만 보고 많은 결정을 내릴 수 있다.
오브젝트 본체에는 접근하지 않고도 GC 작업을 할 수 있게 한다.

---

EInternalObjectFlags

EObjectFlags(오브젝트 자체에 있는 RF_Public, RF_Transient 등)와 별개

EInternalObjectFlags는 GC 전용 내부 상태다

Unreachable        — GC 마크 단계에서 "아직 도달되지 않음"
RootSet            — 이 오브젝트를 루트로 삼아 Reachability 분석을 시작
ClusterRoot        — GC 클러스터의 루트
ReachableInCluster — 클러스터 내에서 도달 가능
Async              — 비동기 로딩 관련

Native             — 엔진 초기화 시점에 C++ 코드가 직접 등록한 오브젝트들

---

Sweep 단계 — GUObjectArray 선형 순회

Sweep은 GUObjectArray를 처음부터 끝까지 선형으로 읽는다


GarbageCollectionKeepFlags = Native | Async | AsyncLoading

for each FUObjectItem in GUObjectArray
    if (item.Flags & Unreachable) && !(item.Flags & GarbageCollectionKeepFlags):
        → BeginDestroy, FinishDestroy, 메모리 해제

---

Mark 단계 — 그래프 순회, GUObjectArray 순회가 아니다

Mark 단계는 다르다. GUObjectArray를 선형으로 훑지 않는다.

1. 루트셋에서 시작: RootSet, Native 오브젝트 등을 작업 큐에 넣음

2. 큐를 소진할 때까지:
    오브젝트를 꺼냄
    → Unreachable 플래그 해제
    → 이 오브젝트의 Reference Token Stream을 실행
       (어떤 offset에 UObject*가 있는지 미리 컴파일된 정보)
    → 발견된 UObject* 들을 큐에 추가
