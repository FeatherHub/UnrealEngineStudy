Crash:
  프로그램이 죽음
  callstack / crash dump가 남음
  어디서 죽었는지 비교적 추적 가능

Hang:
  프로그램이 안 죽고 멈춤
  유저는 강제 종료함
  증거가 거의 안 남음
  개발자는 "가끔 얼어요"만 받음
  ↓
"GameThread가 25초 동안 heartbeat를 갱신하지 못했다"
"마지막으로 X 체크포인트에 있었다"
"그 시점의 전체 thread callstack은 이렇다"
"Present는 멈췄지만 GameThread는 살아있었다"

---

1. UObject shell 생성
   class, outer, name, flags를 맞춰서 객체를 만든다

2. serialized data 읽기
   파일 안의 export serial offset/size 위치로 이동한다

3. UObject::Serialize 호출
   저장되어 있던 property/state를 객체 안에 다시 채운다

4. PostLoad 등 후처리
   로드 완료 상태로 만든다

---

.uasset은 디스크에 있는 바이너리 파일이다.
그 바이너리는 대략 이런 정보를 들고 있다.

1. Package summary
   이 파일이 Unreal package file이 맞는지 확인하는 magic tag
   파일 버전, package flags, package name, 각 테이블의 위치/개수

2. Name table
   파일 안에서 반복해서 쓰는 이름들
   "BP_MyCharacter", "UBlueprint", "GeneratedClass" 등

3. Import table
   이 파일 안의 객체들이 참조하지만, 이 파일이 소유하지 않는 외부 객체들
   SkeletalMesh: /Game/Characters/SK_Mannequin.SK_Mannequin
   부모 클래스: /Script/Engine.Character

4. Export table
   이 파일이 실제로 소유하고 저장하는 UObject 목록 
   실제 데이터 자체라기보다 “이 파일 안에 어떤 객체들이 있고, 어디에 저장 데이터가 있는지”
   Export 0:
     Class      = UBlueprint
     ObjectName = BP_MyCharacter
     Outer      = package root
     SerialOffset = 120000
     SerialSize   = 34000
   Export 1:
     Class      = UEdGraph
     ObjectName = EventGraph
     Outer      = BP_MyCharacter
     SerialOffset = 120000
     SerialSize   = 34000

5. Serialized export data
   Export table에 적힌 각 UObject의 실제 저장 데이터
   property 값, object reference, 배열, 구조체 값 등

6. 기타 데이터
   soft object path
   dependency/preload 정보
   asset registry tag
   editor metadata
   bulk data 위치/참조 (너무 크거나 별도 취급이 필요한 원시 데이터 덩어리)
