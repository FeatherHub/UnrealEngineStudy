SViewport는 Slate 모듈 안에. 

Slate는 Engine을 몰라야 해. 
FRenderTarget, FViewport, RHI, FSceneView — 이런 것들에 의존하면 Slate가 Engine에 묶여버려. 

Slate는 UI 프레임워크이고, 게임 엔진 없이도 에디터 UI나 툴 UI를 구동할 수 있어야 해.


FSceneViewport는 Engine 모듈 안에 있어(Engine/Public/Slate/SceneViewport.h). 
Engine.Build.cs에 Slate 모듈 의존성이 선언되어 있어.

둘 사이의 참조 방향

[Slate 모듈]                    [Engine 모듈]
SViewport
  ISlateViewport*  ←──────────  FSceneViewport
                                  implements ISlateViewport
                                  TWeakPtr<SViewport> ViewportWidget

SViewport는 ISlateViewport 인터페이스 포인터만 들고 있어. FSceneViewport가 뭔지 몰라. 
반대로 FSceneViewport는 SViewport를 TWeakPtr로 들고 있어 — Engine이 Slate를 아는 방향.

ISlateViewport는 이 경계에 놓인 **계약(contract)**이야. 
Slate가 직접 정의하고(Rendering/RenderingCommon.h, SlateCore 모듈), Engine이 구현해. 

이 인터페이스가 Slate 모듈과 다른 모듈들 사이의 경계를 정의해준다.

GetLifetimeReplicatedProps
Replicated Properties = UPROPERTY(Replicated)

이 클래스의 replication layout(schema)를 만드는 기초 재료

RepLayout.cpp 흐름을 보면 엔진은 class replication layout을 만들 때 CDO에게 묻습니다.

TArray<FLifetimeProperty> LifetimeProps;

UObject* Object = InObjectClass->GetDefaultObject();
Object->GetLifetimeReplicatedProps(LifetimeProps);

Class Default Object에게 물어서 class-level replication schema를 만듭니다

이 함수가 만들어야 하는 것은 “지금 이 순간 보낼 목록”이 아니라
이 class의 replication property schema 

Replicate 되어야 할 목록만 따지면 UPROPERTY(Replicated) 로 충분하지만
Lifetime Condition 의 설정까지 유연하게 처리하려면 GetLifetimeReplicatedProps의 형태가 필요
