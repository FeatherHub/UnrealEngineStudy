컴파일된 Vertex Shader는 대략 이렇게 알고 있습니다.

ATTRIBUTE0을 Position으로 해석한다
ATTRIBUTE1을 Tangent로 해석한다

하지만 다음 정보는 알지 못합니다.

Position은 어느 GPU Buffer에 있는가?
그 Buffer의 시작 위치는 어디인가?

이것을 런타임 FVertexFactory가 제공합니다.

FLocalVertexFactory [특정 Static Mesh LOD의 인스턴스]
├─ Type             : Local Vertex Factory
├─ Position Stream  : PositionVertexBuffer
├─ Tangent Stream   : StaticMeshVertexBuffer
├─ UV Stream        : StaticMeshVertexBuffer
├─ Color Stream     : ColorVertexBuffer
├─ Declaration      : 각 Stream을 Shader ATTRIBUTE에 대응
└─ Uniform/Resources

Depth Pass나 Shadow Depth Pass는 흔히 Position만 필요합니다.

Position이 다른 정보와 한 구조체에 뒤섞여 있다면 GPU는 Position만 필요해도 불필요한 Tangent와 UV까지 함께 읽습니다

Attribute는 Shader가 입력을 받는 논리적인 슬롯

Position : ATTRIBUTE0
TangentX : ATTRIBUTE1
TangentZ : ATTRIBUTE2
Color    : ATTRIBUTE3
UV       : ATTRIBUTE4

---

GPU Buffer
   │
   │ Stream
   ▼
Buffer 안의 특정 byte 영역
   │
   │ Component
   ▼
Shader의 ATTRIBUTE 슬롯

---

GPU가 Vertex N의 특정 Component를 찾는 공식

StreamOffset: Buffer 전체에서 이 Stream이 시작되는 위치
Offset: Vertex 레코드 안에서 Component가 위치한 곳
Stride: 다음 Vertex의 레코드로 가기 위한 간격
Type: 그 byte를 Float3, Half2, Color 등 무엇으로 해석할지
