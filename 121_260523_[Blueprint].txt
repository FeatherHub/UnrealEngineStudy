Decoupling 의 종류

Structural Decoupling 
발신자는 수신자의 정체성(Identity)을 모른다. Delegate Signature 계약으로만 연결(Binding)된다.
반면 수신자는 발신자를 추상적으로 참조할 수도 있고, 간접적으로 참조할 수도 있다
  e.g. FindComponentBy 등의 쿼리로 해당 Delegate를 가진 Component에 접근

Cardinality Decoupling
수신자는 없을 수도 있고 1명 일수도 있고 여러 명일 수도 있다. (= 컴파일 타임에 모른다.)
수신자 목록이 런타임에 동적으로 결정된다.


Temporal Decoupling
발신자가 수신자의 실행 완료를 기다리지 않는다.
비동기.

그 포인터가 부모 클래스를 표현하는 다른 UClass 인스턴스를 가리킨다


BP_Action_ProjectileMagic_C   ← (UBlueprintGeneratedClass 타입의 객체)
   .SuperStruct → URogueActionBase의 UClass 인스턴스
        .SuperStruct → UObject의 UClass 인스턴스
             .SuperStruct → nullptr

---

BP_Action_ProjectileMagic_C

그 객체 자체의 C++ 타입: UBlueprintGeneratedClass 
그 객체가 표현하는 클래스

---

class Derived : public Base { int extra; };
// 메모리: [Base 부분][extra]
// Base* p = &derived; 가 안전한 이유.

---

NewObject<URogueActionBase>(this, BPClass)이 실행될 때:

1 
BPClass->GetStructureSize()로 전체 크기 (C++ 부모 부분 + BP 추가분) 조회 후 메모리 할당

2
C++ 부모 클래스의 생성자가 실제로 호출된다 — URogueActionBase::URogueActionBase()가 그 메모리의 앞부분을 초기화 (vptr 셋업 포함)

3
CDO에서 BP-추가 멤버들의 default 값을 복사

4
URogueActionBase*로 반환
