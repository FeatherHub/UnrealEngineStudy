Text Formats

ASCII 
Characters between 32 and 126 inclusive, and 0, 9, 10, and 13
P4 type text
P4 trigger on check-in


ANSI 
ASCII + current codepage
stored as binary on the P4 server


UTF-8
A string made up of single bytes
non-ANSI characters
superset of ASCII
P4 type Unicode


UTF-16
A string made up of 2 bytes
with a BOM
P4 type UTF-16
validated with a P4 trigger on check-in

UE Internal String Representation


stored in memory in UTF-16 format
as FStrings or TCHAR arrays


Most code assumes 2 bytes is one codepoint
so only the Basic Multilingual Plane (BMP) is supported
UCS-2


stored in the endian-ness appropriate for the current platform


strings with all TCHAR characters
less than 0xff are stored as 8-bit
otherwise as 2-byte UTF-16 strings

---

appLoadFileToString() - entry
appBufferToString() - main work


This function recognizes the Unicode byte-order-mark (BOM) in a UTF-16 file 
and if present, will load the file as UTF-16 in either endian-ness.


What happens when the BOM is not present depends on the platform.


On Windows, it will attempt to convert the text to UTF-16 
using the default Windows MBCS encoding and uses MultiByteToWideChar


If this conversion fails on platforms other than Windows
just read each byte and pad it to 16-bits to make an array of TCHAR.


Note
there is no code to detect or decode UTF-8 encoded text files loaded with appLoadFileToString()

---

appSaveStringToFile()

Strings with all TCHAR characters representable by a single byte will be stored as a series of 8-bit bytes
otherwise as UTF-16 unless the bAlwaysSaveAsAnsi flag is passed in
