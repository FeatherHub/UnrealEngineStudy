동일성 기반(장인)을 택하면:
개체가 자신의 상태를 소유한다
세계가 개체들의 집합이다
개체는 세계 안에서 행위한다
변화는 개체의 내부에서 일어난다

메모리: 개체마다 하나의 연속된 구조
접근: 포인터 → 개체 → 상태

묶음 기반(공장)을 택하면:
상태들이 개체를 구성한다
세계가 상태들의 집합이다
변화는 상태에 직접 가해진다
개체는 상태들의 일시적 패턴이다

메모리: 상태마다 하나의 연속된 배열
접근: 인덱스 → 여러 배열 → 상태

`UInputComponent`는 **Pawn의 컴포넌트**다.
Pawn이 교체될 때 입력 바인딩도 함께 교체되어야 하기 때문이다.

PlayerController가 살아있는 동안:
    처음엔 SoldierCharacter를 빙의  → 총 쏘기, 달리기 바인딩
    나중엔 VehicleCharacter를 빙의  → 가속, 핸들 바인딩

입력 바인딩이 PlayerController에 고정되어 있다면 
Pawn이 바뀔 때마다 PlayerController의 바인딩을 수동으로 해제하고 재설정해야 한다

InputComponent가 Pawn에 속하면 UnPossess 시 자동으로 사라지고, 
새 Pawn을 빙의할 때 `SetupPlayerInputComponent`가 다시 호출되어 자동으로 재설정된다

PlayerController는 **입력이 어디로 가는지**를 결정한다. 
Pawn은 **입력이 왔을 때 무엇을 하는지**를 결정한다.

PlayerController: "지금 이 입력을 Pawn에게 전달할 것인가, 
                   아니면 UI에게 전달할 것인가, 
                   아니면 무시할 것인가"

Pawn/Character:   "MoveForward 입력이 왔을 때 
                   나는 앞으로 이동한다"

// 게임 중 — 입력이 Pawn으로 간다
PlayerController->SetInputMode(FInputModeGameOnly());

// UI 열렸을 때 — 입력이 UI로 간다
PlayerController->SetInputMode(FInputModeUIOnly());

직렬화되는 것과 아닌 것이 섞여 있다.

UPackage에 직렬화되는 것
├── UWorld 자체의 일부 설정값    WorldSettings 등
├── ULevel (PersistentLevel)     배치된 Actor들, 지오메트리
└── NavMesh 베이크 데이터        UNavigationSystem이 생성한 결과물
                                 .umap에 구워져서 저장됨

직렬화되지 않는 것 (런타임에 생성)
├── FPhysScene      UObject 아님, 직접 메모리, 매번 새로 생성
├── UNetDriver      Transient, 런타임 연결 상태
├── FTickTaskManager UObject 아님
├── FTimerManager   UObject 아님
└── UAISystem       런타임 상태, 직렬화 안 됨
