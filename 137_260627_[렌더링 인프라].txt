command list

"Draw할 때 constant buffer 내용을 0xABC에서 읽어라"

Renderer::WaitForGpuCompleteFrame(frameFenceValue)
{
  commandQueue->Signal(fence.Get(), frameFenceValue);

  if (fence->GetCompletedValue() < frameFenceValue)
  {
    // "fence 값이 frameFenceValue 이상이 되면 fenceEvent를 켜줘."
    fence->SetEventOnCompletion(frameFenceValue, fenceEvent)

    WaitForSingleObject(fenceEvent)
  }
}

병원 환자 차트

접수에서 차트 개설(이름·생년 = 카메라 정보) = FSceneView 복사로 생성

분류 → 채혈·영상의학 → 진단, 각 스테이션이 같은 차트에 결과를 적는다 = 가시성② → 수집③ → UB④ → 패스⑤ 가 차례로 채움

차트는 환자를 따라다니며 모두가 읽고 쓴다 = FSceneRenderer가 계속 쥐고, 각 단계가 읽고 쓴다

결과물은 차트가 아니라 치료다 = 차트의 일부(처방전 = ViewUniformBuffer)만 약국(GPU)에 전달되고, 
퇴원 요약만 평생 의무기록(FSceneViewState)에 남는다

퇴원하면 작업용 차트는 폐기 = 프레임 끝에 FViewInfo 소멸

---

같은 수명. 전부 1프레임 살고 같은 아레나(FSceneRenderingBulkObjectAllocator/SceneRenderingAllocator)에서 나서 같이 죽는다. 
같은 수명 데이터를 한 객체에 모으면 할당·해제·소유가 한 번에 끝난다.
수명 버킷.
