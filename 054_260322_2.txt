Newton의 힘 언어

variational principle과 Lagrange formalism이라는 구조 언어

Conservation
현상을 지역적으로 보지 말고 전역적으로 보라
“변하지 않는 양을 잡아라”는 접근으로 이동
momentum, angular momentum, energy
center of mass
다체 문제를 “집합의 대표점”으로 압축하는 강력한 장치

Oscillation
simple harmonic motion, damping, driving, resonance, Fourier series
물리학의 모형실
많은 시스템이 equilibrium 주변에서는 oscillator처럼 보인다
줄, 분자, 회로, 구조물, 빛의 mode
복잡성을 mode들의 합으로 분해하는 시각
Fourier series
아무 복잡한 주기적 입력도 많은 단순 진동의 합으로 쪼갤 수 있다

Variational viewpoint
자연은 ‘그냥 움직이는’ 것이 아니라 어떤 조건을 만족하며 움직인다
constraints, generalized momenta, ignorable coordinates
힘 목록을 써 넣는 기술
허용 가능한 모든 경로 중에서 실제 경로를 고르는 원리

Geometry of orbits
두 물체 문제는 우주 역학의 문법이다
상호작용의 기하학
문제의 자유도를 줄여 핵심만 남기는 전형적인 물리학적 추상화

점입자의 회전과 시점 변환
inertia tensor는 “회전판에서의 mass”
회전에서는 물체가 각 방향에 대해 얼마나 회전에 저항하는지 행렬 형태로 알아야 해요
스칼라 하나로는 방향 의존성을 담을 수 없기 때문에 구조가 커지는 거예요.

normal mode
시스템 고유의 mode
적절한 basis로 좌표계를 갈아끼우는 일

dW=F⋅dr

mass는 물체의 고유한 inertia
weight는 gravitational field 안에서 받는 force

Force
push or pull

움직이는데 가속하지 않으면 friction 같은 상쇄 force를 추론해요.

force는 운동 변화의 원인을 해석하며 드러나는 quantity예요.

어떤 reference frame이 inertial frame인가를 판별하는 기준

Second Law
위치 r만으로는 운동 상태가 안 정해져요.
속도도 알아야 하고, 더 나아가 그 속도가 어떻게 바뀌는지도 알아야 해요.

net force는 motion의 “변화율”을 만들고,
mass는 그 변화에 저항하는 계수다.

모든 데이터 블록이 DNA 구조체로 정의되어 있고,
컴파일 시점에 이 구조체들의 메타데이터(필드 이름, 타입, 오프셋)가 
바이너리에 내장됨

덕분에 다른 버전의 Blender로 저장된 파일도 
구조체 레이아웃 차이를 런타임에 감지해서 
자동으로 마이그레이션할 수 있음

RNA는 DNA 위에 올라오는 리플렉션 레이어
C 구조체의 필드들을 이름으로 접근 가능하게 만들고, 
각 프로퍼티에 범위/기본값/UI 힌트 같은 메타데이터를 붙임. 
Python API와 UI 위젯이 실제 C 구조체를 직접 알 필요 없이 
RNA를 통해 데이터에 접근함

bpy.data.objects["Cube"].location.x = 1.0 같은 Python 코드가 
RNA를 통해 C 메모리를 건드리는 것.

모든 데이터 블록은 Main 구조체가 소유하는 링크드 리스트로 관리됨. 
씬 전체가 Main 하나에 들어있고, 
파일 저장/불러오기는 이 Main을 직렬화/역직렬화하는 것.

오브젝트 A의 위치가 바뀌면, 
A를 참조하는 컨스트레인트, A 위에 올라온 모디파이어, A를 보고 있는 카메라의 frustum 등 
수많은 것들이 갱신되어야 함. 
이것을 올바른 순서로, 가능하면 병렬로 실행해야 함.

DNA 데이터 블록들 사이의 참조 관계를 분석해서 DAG를 구성함. 
노드는 "평가 단위(ComponentNode)"이고 
에지는 "이 노드가 갱신되려면 저 노드가 먼저 갱신되어야 한다"는 의존성. 
컴포넌트 단위로 더 세분화되어 있어서 
오브젝트 전체가 아니라 "이 오브젝트의 지오메트리 컴포넌트"만 
dirty로 표시하는 것이 가능.

씬에서 변경이 발생하면 해당 노드를 dirty로 표시하고, 
뷰포트 갱신/렌더 시점에 dirty 노드들을 위상 정렬 순서로 평가. 
변경되지 않은 노드는 캐시된 결과를 재사용(Caching). 
의존성이 없는 노드들은 병렬 평가.

Depsgraph는 원본 DNA 데이터를 수정하지 않고 
"평가된 복사본(Evaluated Copy)"을 만들어냄. 
뷰포트와 렌더러는 이 Evaluated 데이터를 보고, 
편집 오퍼레이션은 Original 데이터에 가해짐. 
비파괴 워크플로우가 성립하는 근거.

poll()은 "이 오퍼레이터가 현재 컨텍스트에서 실행 가능한가"를 반환
메뉴/단축키가 자동으로 비활성화되는 근거가 여기에 있음. 
invoke()는 사용자 인터랙션을 처리(모달 드래그 등), exec()는 실제 연산을 수행.
