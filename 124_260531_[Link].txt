MSVC 맹글링 규칙으로 인코딩된 시그니처

Q — public 비가상 멤버 함수
E — x64 포인터 한정자
A — non-const non-volatile 인스턴스
X — void 반환
H — int 파라미터
@Z — 파라미터 목록 끝

컴파일러가 .obj에 심볼을 집어넣을 때 이 맹글드 이름으로 집어넣고, 링커는 이 이름으로 검색한다. 
타입이 달라서 overload가 다르면 맹글드 이름이 달라지므로 서로 다른 심볼로 취급된다.

"__declspec(dllimport)"

해당 함수나 클래스가 현재 빌드 중인 모듈 내부가 아니라, 
외부에 분리된 다른 "dll" 파일로부터 가져와야 하는 것임을 나타내는 컴파일러 키워드

빌드 시점에 링커가 참고하는 것은 lib 파일

정적 링크의 lib
  출력 실행파일에 코드가 복사됨

동적 링크의 lib
  import library

컴파일러(MSVC에서는 cl.exe)는 .cpp 파일 하나를 받아서 .obj 파일 하나를 만든다. 
1:1 대응이다. 이 .cpp 하나가 컴파일의 독립적인 단위이고, 이걸 번역 단위(translation unit, TU) 라고 부른다

RoguePlayerController.cpp  →  cl.exe  →  RoguePlayerController.cpp.obj
Module.ActionRoguelike.cpp →  cl.exe  →  Module.ActionRoguelike.cpp.obj

링커에게 .cpp는 존재하지 않는다. 링커가 보는 단위는 .obj다.
