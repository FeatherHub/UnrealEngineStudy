물리 법칙이란
“현재 상태 → 다음 상태”의 규칙이기 때문입니다.
(C) 고전역학에서 상태의 구체적 내용
질점 하나의 경우
고전역학에서 최소 상태는:
(r, v)

r : 위치 (position)
v : 속도 (velocity)

왜 이 둘인가?

위치만 알면 → ❌ (어디로 가는지 모름)
속도만 알면 → ❌ (어디 있는지 모름)
둘 다 알면 → ⭕
→ 뉴턴 방정식으로 가속도 계산 가능

즉,
상태 = 미분방정식을 닫히게 만드는 최소 변수 집합
(D) 그래서 상태의 “타입”이 그렇게 생겼다
질점 하나, 3차원 공간이라면:

위치 : ℝ³
속도 : ℝ³

따라서 상태 공간은
ℝ³ × ℝ³ = ℝ⁶
이건 결코 우연이 아닙니다.
자유도 × (좌표 + 변화율)
→ 이것이 고전역학 상태의 보편적인 형태입니다.
(E) “왜 이런 타입이냐?”의 진짜 이유
상태의 타입은 다음 세 가지가 결합되어 결정됩니다:

미분방정식의 차수
2차 미분방정식 → 위치 + 속도(1차 미분) 필요
관성 법칙 (뉴턴 제1법칙)
외력이 없으면 속도가 일정하게 유지된다
예측 가능성의 요구
현재 정보만으로 미래를 (이론적으로) 완전히 계산할 수 있어야 한다

이 세 조건이 합쳐지면서
다음과 같은 구조가 강제됩니다:
state = (좌표, 좌표의 시간 미분)
즉, state = (r, dr/dt)
(F) 타입 관점으로 다시 쓰면 (TypeScript 스타일)
TypeScripttype Vec3 = [number, number, number];

type Position = Vec3;
type Velocity = Vec3;

type State = {
  r: Position;   // 위치
  v: Velocity;   // 속도
};
물리 법칙은 반드시 다음 형태여야 합니다:
TypeScriptfunction evolve(state: State, dt: number): State {
  // 다음 상태를 현재 상태와 시간 간격 dt만으로 결정
}
이 구조가 깨지면 발생하는 문제:

예측 불가능해짐
시간에 대한 결정론적 진화가 붕괴
더 이상 물리 법칙이라고 부를 수 없음
