Physics 시뮬레이션은 결정론적 고정 timestep으로 돌아야 하고
Game Thread는 가변 프레임레이트로 돌아

FBodyInstance

Chaos Solver 레이어
PBD (Position Based Dynamics) 방식으로 constraint를 푼다
PhysX (impulse-based)와의 주요 차이

PBD는 위치를 직접 보정하는 방식이라 안정성이 높고 대규모 rigid body에 유리
대신 에너지 보존이 impulse 방식보다 덜 정확

Geometry / Collision 레이어
충돌 형상은 FKAggregateGeom으로 표현
(sphere, box, capsule, convex hull의 집합)
런타임에는 Chaos의 implicit geometry 표현으로 변환

Collision query (raycasting, sweep, overlap)는 
FPhysicsInterface를 통해 들어오고, 
결과는 FHitResult로 Game Thread에 반환

[Game Thread]
  AActor::Tick()
  → UPrimitiveComponent transform 변경
  → FBodyInstance에 변경 사항 기록 (dirty)

[Physics Thread Kick]
  FPhysScene::StartFrame()
  → dirty body들을 Physics Thread에 push (marshal)
  → Chaos solver tick (고정 substep, e.g. 1/60s)
  → 충돌 감지 → constraint resolution → 위치 적분

[Game Thread Sync]
  FPhysScene::EndFrame() / WaitPhysScenes()
  → Physics Thread 결과를 Game Thread로 pull back
  → FBodyInstance가 결과를 UPrimitiveComponent에 반영
  → AActor transform 업데이트

이 마샬링 구조 때문에 물리 결과가 1 프레임 지연되는 건 피할 수 없는 구조적 특성이야. 
이게 눈에 띄는 경우가 있어서 Async Physics 모드와 일반 모드를 선택적으로 쓸 수 있어.

조합 폭발이 발생하는 구조
O(n²) 충돌 쌍 검색

Broadphase
ISpatialAcceleration<FAccelerationStructureHandle> 인터페이스로 추상화
기본 구현은 TAABBTree

모든 파티클의 AABB를 계층적 트리로 관리
O(n log n)

TBoundingVolumeClusters는 공간적으로 근접한 sleeping body들을 그룹으로 묶는다. 
클러스터 전체가 다른 body와 overlap되지 않으면 내부는 아예 검사 안 해
sleeping body가 많은 large-scale destruction에서 핵심적인 최적화.

Narrowphase: GJK / EPA / SAT

Broadphase가 넘겨준 overlap 후보 쌍들 각각을 정밀하게 검사하는 단계.

GJK (Gilbert-Johnson-Keerthi)
두 convex shape 사이의 최소 거리
두 shape의 Minkowski difference가 원점을 포함하는지를 
iterative simplices 방법으로 검사

GJK가 penetration을 감지하면
EPA (Expanding Polytope Algorithm)로 전환해서 
침투 깊이와 접촉 법선을 계산한다.

Box-box에는 SAT (Separating Axis Theorem)가 별도 fast path로 존재
특정 primitive pair에 특화된 알고리즘이 더 빠르기 때문.

Solver & Constraints: 물리 법칙이 집행되는 곳
PhysX는 impulse-based 방식
충돌을 감지하면 운동량 보존에 따라 impulse를 계산해서 속도를 직접 수정
Chaos는 Position Based Dynamics (PBD)

PBD에서 constraint는 C(x) = 0 형태의 기하학적 조건
솔버는 이 조건을 만족시키도록 위치 x를 반복적으로 투영
속도는 위치 변화로부터 역산

impulse-based 방식은 수치 정밀도가 높지만 
constraint가 많아지면 (대규모 destruction처럼 수천 개의 접촉) 
시스템이 불안정해진다

PBD는 iteration 수를 줄이면 constraint가 덜 정확하게 풀리지만 
시스템이 발산하지 않고 안정하게 수렴한다

TPBDRigidsEvolution이 실제 적분 엔진
FPBDRigidsSolver가 외부 인터페이스를 제공

kinematic body들의 목표 위치를 적용
중력, RadialForce 등 외력을 모든 dynamic particle에 누적
symplectic Euler 적분으로 새 위치를 계산 (속도 먼저 업데이트, 업데이트된 속도로 위치 적분)
FContactPoint 목록을 생성해 FPBDCollisionConstraints에 등록
모든 constraint를 N회 반복해서 투영 (UPhysicsSettings에서 설정)
위치 변화에서 속도를 역산
에너지가 임계값 이하인 island를 sleeping으로 전환

Constraint 종류
FPBDCollisionConstraints가 contact manifold를 관리
restitution (반발계수), friction을 constraint coefficient로 인코딩. 
XPBD (Extended PBD)로 compliance 파라미터를 추가해서 restitution 튜닝 가능

FPBDJointConstraints가 물리적 조인트를 처리
각 joint는 6DOF (position 3 + rotation 3) constraint의 부분집합
Hinge joint는 rotation 2축을 잠그고 1축만 열어두는 식
FJointSolverGaussSeidel이 각 joint를 Gauss-Seidel iteration으로 풀어

모든 코드 구조물에는 특정 속성들이 항상 붙어있다
소유자가 있고, 호출자가 있고, 실행되는 컨텍스트가 있고, 실패 방식이 있다

코드가 반드시 함축하고 있는 속성들을 읽어내는 행위다

이 함수가 사용하는 리소스는 누가 소유하는가
이 함수가 생성하는 것은 누가 소유할 것인가

이 클래스는 무엇의 소유자인가
이 클래스 자신은 누구의 소유물인가

소유권이 없으면 유효성을 보장할 주체가 없어진다

FVector::Dot(const FVector& A, const FVector& B)
A · B = |A||B|cosθ
두 벡터가 같은 방향이면 양수, 수직이면 0, 반대 방향이면 음수.
단위 벡터끼리의 내적은 cosθ
B 방향으로 A를 투영한 길이에 |B|를 곱한 값

FVector::Cross(const FVector& A, const FVector& B)
A × B = |A||B|sinθ * n̂
n̂은 A와 B 둘 다에 수직인 단위 벡터. 방향은 오른손 법칙
결과 벡터의 크기는 A와 B가 만드는 평행사변형의 넓이
두 벡터에 수직인 법선 벡터 계산 (삼각형 법선, 평면 법선)

교환법칙이 성립하지 않는다
이게 외적의 특징이자 방향 정보가 담겨있는 이유다
A × B와 B × A는 부호가 반대

UE는 왼손 좌표계가 아닌 Z-up 오른손 좌표계를 사용하지만
관습적으로 X가 Forward, Y가 Right, Z가 Up이다
DCC 툴과 좌표 변환이 필요한 이유
FBX 임포트 시 Z-up ↔ Y-up 변환이 자동으로 일어난다

FRotator — 인간을 위한 회전 표현
struct FRotator {
    double Pitch;  // Y축 회전 (위아래 고개)
    double Yaw;    // Z축 회전 (좌우 몸통)
    double Roll;   // X축 회전 (몸통 기울임)
};

Euler angles
회전을 세 개의 독립된 축 회전의 합성으로 표현
인간이 이해하고 에디터에서 편집할 수 있는 형태의 회전 표현
Pitch를 30도로 설정해라
vs
쿼터니언의 (W=0.966, X=0, Y=0.259, Z=0)

UE의 FRotator는 Yaw → Pitch → Roll 순서로 적용된다
(intrinsic 기준)

Gimbal Lock
Euler angles의 구조적 결함
Pitch가 ±90도에 가까워지면 Yaw와 Roll이 같은 회전을 표현하게 되어 한 자유도를 잃는다
조종사가 완전히 수직으로 기울었을 때 좌우 회전과 몸통 기울임이 구분되지 않는 현상

FQuat — 회전의 실제 내부 표현
struct FQuat {
    double X, Y, Z, W;
};
축-각도 표현
W = cos(θ/2)
X = sin(θ/2) * n̂.x
Y = sin(θ/2) * n̂.y
Z = sin(θ/2) * n̂.z

4D 구면(S³)에서의 투영

단위 쿼터니언 (|Q| = 1)은 3D 회전과 일대일 대응한다
Q와 -Q가 같은 회전을 표현한다
S³에서의 antipodal points가 SO(3)의 같은 원소로 매핑된다
쿼터니언이 회전군 SO(3)의 double cover인 이유다

FQuat::Dot(A, B) < 0이면 같은 방향을 향하지만

행렬 곱과 마찬가지

복소수가 2D 회전을 완벽하게 표현한다
3D에 대해서도 같은 걸 할 수 있는가?
방법은 순전히 대수적 조작과 기하학적 해석의 왕복
3개의 성분으로 이루어진 수 체계 (i, j, k를 가진 삼원수)를 시도했는데, 곱셈이 성립하지 않았어. 
결합법칙이나 분배법칙을 만족하는 방식으로 i, j, k의 관계를 정의할 수가 없었지.
차원을 하나 더 올려야 한다, 4개의 성분
그리고 i² = j² = k² = ijk = -1

3D 공간 모형을 만들어서 붙잡은 게 아니라, 
2D 복소수에서 작동하는 구조를 대수적으로 확장하는 감각

the dot product doesn't change under rotation
eigenvalues are invariant under similarity transformations
the determinant measures something geometric that also survives these changes.
Eigenvectors reveal the natural axes along which a transformation acts

Rotations themselves form a Lie group
you can't simply add two rotations, but you can compose them smoothly, 
and this curved structure means angular velocity lives in the tangent space 
while rotations themselves live on the manifold

The dot product lets us identify vectors with linear functionals on vectors, 
which is why normals transform differently than positions
they're actually elements of the dual space, 
and this is what the transpose-inverse rule for normals is really capturing.

---

matrices become dual
either active transformations rotating the object, 
or passive ones rotating the coordinate system

This duality explains why the view matrix inverts the camera's transformation.

능동적(active) 해석
공간 속의 물체를 실제로 이동시키는 것
벡터 v를 받아서 다른 벡터 Mv를 돌려주는 것. 
로봇 팔이 회전하는 것.

수동적(passive) 해석
물체는 그대로이고 좌표계를 바꾸는 것
내가 카메라를 오른쪽으로 이동하면, 세계의 모든 것이 왼쪽으로 이동한 것처럼 보여. 
물체는 안 움직였는데 좌표가 바뀌었어

이 두 해석은 수학적으로 정확히 역(inverse) 관계야
물체를 행렬 M으로 변환하는 것은, 좌표계를 M⁻¹으로 변환하는 것과 동일한 결과를 만들어

이것이 UE의 뷰 행렬(View Matrix)이 카메라 트랜스폼의 역인 이유야

카메라를 오른쪽으로 이동시키는 변환 T가 있을 때,
같은 카메라 이동을 "세계가 왼쪽으로 이동"으로 표현하는 뷰 행렬은 T⁻¹이야

이 이중성을 이해하지 못하면, "왜 뷰 행렬에 역행렬을 쓰는가"가 영원히 패턴 암기로 남아.

불변량 — 변환이 파괴하지 못하는 것들
변환을 이야기했으면, 반드시 물어야 할 것이 있어 — 무엇이 보존되는가?

기하학은 변환들의 집합이 아니라
그 변환들이 보존하는 것들의 집합

내적(dot product)의 스칼라 값은 회전에 대해 불변이야
벡터의 좌표는 기저가 바뀌면 달라지지만
두 벡터의 내적값은 달라지지 않아
왜? 회전은 거리와 각도를 보존하는 변환이고, 내적은 정확히 거리와 각도를 담는 구조니까.

행렬식(determinant)은 아핀 변환에 대해 부분적으로 불변이야
두 행렬의 행렬식의 곱이 두 행렬 합성의 행렬식이 되는 성질 — det(AB) = det(A)det(B)

물리학에서 "좌표계에 무관한 물리량"이 불변량이야
어떤 관측자가 어떤 좌표를 쓰든 동의해야 하는 것들.
에너지, 전하, 질량

불변량이 없으면 두 표현이 같은 것을 말하고 있는지 알 수 없어.

---

행렬식 — 공간이 변환에 의해 얼마나 뒤틀리는가
단위 정육면체를 행렬 M으로 변환했을 때 만들어지는 평행육면체의 부피
이것이 |det(M)|이야
det = 2라면 변환 후 부피가 2배. 
det = 0이라면 변환이 공간을 더 낮은 차원으로 찌부러뜨린 것

부호는 방향성(orientation)을 담아. 
det > 0이면 변환이 오른손 좌표계를 오른손 좌표계로 보내. 
det < 0이면 거울 반사가 포함돼

이것이 UE에서 물체를 음수 스케일로 설정했을 때 법선이 뒤집히는 이유야. 
거울 변환이 orientation을 뒤집기 때문이야

det(AB) = det(A)det(B)는 이제 공식이 아니라 당연한 사실이야. 
"A 변환으로 부피가 k배가 되고, B 변환으로 m배가 된다면, 연속 적용하면 km배가 된다."

어떤 변환에는 특별한 방향이 있어. 변환했을 때 방향은 유지되고 크기만 바뀌는 방향
이 방향이 고유벡터(eigenvector)이고, 배율이 고유값(eigenvalue)이야.
Mv = λv
"이 벡터 v는 M에 의해 λ배 스트레칭될 뿐, 회전되지 않는다."

회전 행렬의 고유값을 계산하면 복소수가 나와
순수한 회전에는 실수 고유벡터가 없어
방향을 유지한 채 크기만 바뀌는 방향이 존재하지 않아. 
회전은 모든 방향을 틀어버리거든
이것이 회전이 기하학적으로 특별한 이유의 또 다른 표현이야.

반대로 대칭 행렬(symmetric matrix)은 항상 실수 고유값과 서로 수직인 고유벡터들을 가져
물리학에서 관성 텐서가 대칭 행렬인 이유가 여기 있어
관성 텐서의 고유벡터들이 물체의 주축(principal axes), 
즉 회전시켜도 흔들리지 않는 자연스러운 회전축이야

---

회전의 공간은 왜 "굽어" 있는가
두 회전을 "더하는" 것이 의미가 없어. 
회전 A와 회전 B를 성분별로 더하면 회전이 아닌 다른 행렬이 나와.
회전은 합성(composition) 되지, 더해지지 않아.

이것이 회전의 집합이 벡터 공간이 아닌 이유야
벡터 공간은 덧셈에 대해 닫혀 있어야 해.
회전의 집합은 합성(곱셈) 에 대해 닫혀 있어

두 회전을 합성하면 또 다른 회전이야. 
역이 있어 (역회전). 항등원이 있어 (0도 회전). 결합법칙이 성립해. 
이게 군(group) 의 구조야.

그런데 회전의 군은 여기서 멈추지 않아
회전을 "조금씩 연속으로" 할 수 있어
0도에서 시작해서 90도까지 매끄럽게 회전하는 것.
이 매끄러움이 회전의 군을 단순한 군이 아니라 
Lie group 으로 만들어
군이면서 동시에 매끄러운 다양체(manifold)인 것

회전 자체 — SO(3)의 한 원소 — 는 구부러진 공간(manifold) 위에 있어
그 공간의 어떤 한 점 (예: 항등 회전, 즉 "아무것도 안 한 상태")에서의 접평면(tangent space)은 완전히 평평한 벡터 공간이야
이 접평면이 **Lie algebra**야.
각속도는 리 대수에 산다

접평면에서 다양체로 올라가야 해.
이것이 지수 사상(exponential map)
FQuat::Exp()가 그것이고, 역이 FQuat::Log()

기저 변환 — 같은 실재를 다른 언어로 말하기
기저가 2배 커지면 좌표는 1/2이 되어야 같은 벡터를 표현해. 
기저와 좌표는 역방향으로 움직여. 이것이 반변(contravariant)이야.

그런데 법선 벡터(normal vector)는 다르게 움직여
기저가 2배 커지면 법선도 2배가 되어야 해
기저와 같은 방향으로 움직여. 이것이 공변(covariant)이야

왜 법선이 다르게 움직이는가? 법선은 평면을 정의해 
"이 평면에 수직인 방향"
평면 자체가 기저의 변화에 어떻게 반응하는지 생각해보면, 
법선은 다른 방식으로 변환되어야 한다는 게 기하학적으로 분명해져
법선은 다른 종류의 수학적 객체 — 이중 공간(dual space)의 원소 — 이기 때문이야

법선은 벡터가 아니라 코벡터(covector)다
벡터가 화살표라면, 코벡터는 평행한 층(layer)이다.

지형도의 등고선을 생각해봐
등고선은 방향이 없어
그런데 벡터(이동 방향)가 등고선을 몇 개 가로지르는지 세면 — 그것이 내적이야
벡터와 코벡터가 짝을 이루면 스칼라가 나와

A·B를 "벡터 B를 코벡터 A에 집어넣어 스칼라를 얻는 행위"로 읽을 수 있어
내적 공간(inner product space)은 벡터와 코벡터 사이에 자연스러운 동형사상을 제공하는 구조야 
공간이 non-uniform하게 늘어날 때 (non-uniform scale, curved space) 이 구분이 터져 나와.

det(AB) = det(A)det(B)인 이유
"A 변환으로 부피가 k배, B 변환으로 m배가 된다면, 연속 적용하면 km배다." 자명해.

det < 0

내적은 세 가지 다른 질문에 동시에 답한다. 
그리고 이 세 질문이 같은 연산으로 수렴한다는 것이 Euclidean 공간의 특별함이다.

A와 B는 얼마나 같은 방향인가? → A·B의 부호와 크기.
A의 길이는 얼마인가? → A·A = |A|².
B 방향으로 A가 얼마나 들어있는가 (투영)? → A·B̂ (B의 단위벡터와의 내적)

Euclidean 공간은 거리와 각도가 같은 구조에서 나와
이 공간에서 "가깝다"와 "같은 방향이다"는 분리할 수 없어
내적이 그 통합을 표현해

내적이 존재한다는 것은 공간에 metric이 있다는 것이야
Metric이 없는 공간에서는 "거리"도, "각도"도, "수직"도 없어
위상 공간(topological space)이 그런 공간이야
열리고 닫힘만 있지 거리는 없어. 내적을 추가하는 순간 위상 공간이 기하학적 공간이 돼.

직교 분해는 공간을 두 개의 독립된 부분 공간으로 쪼개는 유일한 방법이다
"수직"이 바로 "독립"의 기하학적 표현이다.

벡터 v를 방향 n̂으로 분해한다는 것은: v = (v·n̂)n̂ + (v - (v·n̂)n̂).
첫 번째 항은 n̂ 방향 성분, 두 번째 항은 n̂에 수직인 성분. 이 둘이 내적이 0이라는 것이 핵심이야
수직인 두 성분은 서로에게 완전히 무관해

“두 벡터의 관계를 하나의 숫자로 요약하고 싶은데,
그 숫자가 기하적으로 믿을 만한 의미를 가져야 한다.”

(1) 합에 대해 잘 반응해야 해요
벡터 u+v가 있으면, 그것이 w와 맺는 관계는
u와 w의 관계, v와 w의 관계가 자연스럽게 합쳐진 것이어야 해요.

즉 관계량은 additive해야 해요.
그래서 선형성이 나와요.

(2) 배율에 대해 잘 반응해야 해요
벡터를 두 배 늘리면, 그 벡터가 다른 벡터와 맺는 관계량도 두 배가 되어야 자연스러워요.
즉 관계량은 homogeneous해야 해요.

(3) 자기 자신과의 관계는 길이를 낳아야 해요
벡터 v와 v의 관계량은 음수가 되면 안 돼요.
즉 자기 관계는 크기의 제곱 같은 역할을 해야 해요.

4-1. 길이

||v|| = sqrt(<v,v>)

즉 길이는 별도로 주어지는 것이 아니라,
자기 자신과의 관계값에서 추출돼요

4-2. 각도
관계량 = 크기 × 정렬 정도
u·v = ||u|| ||v|| cos θ

길이 효과는 ||u|| ||v||,
방향 일치 효과는 cos θ예요.

4-3. 투영
dot = 방향 일치도의 측정
dot = 한 벡터 안에 다른 벡터 방향 성분이 얼마나 들어 있는가의 측정

“좌표축별 기여를 따로 계산한 뒤 더할 수 있어야 한다”는 요구

관계량 <u,v>도 각 축 기여의 합으로 계산되어야 자연스러워요.
그렇지 않으면 분해해서 계산하는 선형대수 전체가 무너져요.

실수 공간에서 <u,v> = <v,u>라는 대칭성은
이 관계량이 “u가 v를 보는 값”과 “v가 u를 보는 값”이 같은 종류의 관계량임을 뜻해요

길이와 각의 관계는 본질적으로 상호적이니까요.

u가 v와 이루는 각
v가 u와 이루는 각
은 같은 각이에요.

그래서 dot product는 대칭적이어야 해요
복소수 공간에서는 위상이 개입해서 완전한 대칭 대신 켤레대칭이 필요해져요

"얼마나 같은 방향인가"를 측정하는 함수 ⟨A, B⟩ → 스칼라를 설계한다고 해보자
이것이 측정으로서 의미를 갖기 위해 어떤 조건이 필요한가?

쌍선형성 (bilinearity)
벡터를 2배로 늘리면 측정값도 2배가 되어야 해
⟨2A, B⟩ = 2⟨A, B⟩
스케일에 일관되지 않은 측정은 측정이 아니야
두 벡터를 합쳤을 때 각각의 측정값의 합과 같아야 해

대칭성 (symmetry)
⟨A, B⟩ = ⟨B, A⟩. "A가 B 방향을 얼마나 향하는가"와 "B가 A 방향을 얼마나 향하는가"가 같아야 해
순서가 결과를 바꾼다면, 그것은 방향들 사이의 관계가 아니라 방향에 할당된 무언가가 측정되는 거야
대칭성은 측정의 대상이 "관계" 자체임을 보장해

positive definiteness
⟨A, A⟩ ≥ 0, 등호는 A = 0일 때만
벡터가 자기 자신과 얼마나 같은 방향인가 — 이것이 길이의 제곱이야

---

코시-슈바르츠
positive definiteness이 각도를 낳는다

"길이는 항상 양수다"라는 조건 하나가, "⟨A, B⟩ / (‖A‖‖B‖)는 항상 [-1, 1] 안에 있다"를 강제해
그리고 [-1, 1] 안의 값은 코사인으로 읽을 수 있어

‖A - tB‖² ≥ 0
‖A‖² - 2t⟨A, B⟩ + t²‖B‖² ≥ 0
4⟨A, B⟩² - 4‖A‖²‖B‖² ≤ 0
⟨A, B⟩² ≤ ‖A‖²‖B‖²
|⟨A, B⟩| ≤ ‖A‖·‖B‖

거리와 각도는 하나다 — 편광 항등식 Polarization Identity

거리와 각도는 하나의 구조의 두 얼굴이다
거리만 알아도 각도를 복원할 수 있고, 
각도와 하나의 길이만 알아도 모든 거리를 복원할 수 있다
이 둘은 분리된 개념이 아니라 같은 것을 다른 방향에서 읽는 것이다.

⟨A, B⟩ = ¼(‖A + B‖² - ‖A - B‖²)
두 벡터의 내적은 네 개의 길이 — ‖A+B‖, ‖A-B‖, ‖A‖, ‖B‖ — 로부터 완전히 복원 가능해

내적을 직접 계산하지 않아도, 길이들만 알면 내적을 알 수 있어
내적을 알면 각도를 알 수 있어

거리만 알면 → 내적을 알고 → 각도를 안다.

반대 방향도 마찬가지야

각도와 하나의 길이를 알면 → 내적을 알고 → 모든 거리를 안다

더 일반적인 거리 공간 (metric space)에서는 이 복원이 불가능해
거리가 있어도 각도가 정의되지 않는 공간이 있어
Euclidean 공간은 거리가 내적으로부터 나오는 특별한 경우야

‖A + B‖² + ‖A - B‖² = 2(‖A‖² + ‖B‖²)

---

Euclidean 거리는 가능한 많은 거리 구조들 중 특별한 하나다
Euclidean은 그 중에서 내적에서 유도된, 그리고 모든 점에서 균일한 특수한 경우다

거리 공간 (metric space)의 조건
자기 자신까지의 거리는 0
거리는 대칭적
삼각 부등식: d(A, C) ≤ d(A, B) + d(B, C).

택시 기하학 (격자를 따라 이동하는 거리), 구면 위의 대원 거리, 쌍곡면의 거리

Euclidean 공간의 내적은 어디서나 같아
이것이 "평평함 (flatness)"의 수학적 정의야.

구면 위에서는 내적 구조가 점마다 달라
극점 근방과 적도 근방에서 "거리의 느낌"이 달라
이것이 리만 계량 (Riemannian metric)이야
각 점에서의 접평면에 내적이 있는데, 그 내적이 점에 따라 부드럽게 변해
Euclidean은 이 리만 계량이 모든 점에서 동일한 특수 경우야

벡터 공간에 내적이 없어도 있는 것

이것들은 측정이 아니라 구조의 언어야

내적을 추가하면 새로 생기는 것

변환이 "연속인가"는 말할 수 있지만 "얼마나 이동했는가"는 말할 수 없어.

선형 독립은 내적 없이 있어 → 내적에 의존하지 않아 → 좌표 선택과 무관한 개념이야.

그런데 "얼마나 회전했는가"는 내적이 있어야 말할 수 있어.

임의의 기저에서 좌표 계산은 연립방정식이다
직교 기저에서 좌표 계산은 내적 하나다
이 차이가 직교 기저를 "자연스러운" 기저로 만드는 이유다.

임의의 기저 {v₁, v₂, v₃}가 있어. 벡터 w를 이 기저로 표현하고 싶어:
w = a₁v₁ + a₂v₂ + a₃v₃

a₁, a₂, a₃를 구하려면 세 개의 연립방정식을 풀어야 해. 
기저 벡터들이 서로 얽혀 있기 때문이야

기저가 직교하면: ⟨vᵢ, vⱼ⟩ = 0 (i ≠ j). 이 경우 양변에 vᵢ의 내적을 취하면:
⟨w, vᵢ⟩ = aᵢ⟨vᵢ, vᵢ⟩ = aᵢ‖vᵢ‖²

따라서 aᵢ = ⟨w, vᵢ⟩/‖vᵢ‖². 
연립방정식 없이 내적 하나로 각 성분이 독립적으로 나와.

기저가 정규 직교(orthonormal)하면 ‖vᵢ‖ = 1이니까 
aᵢ = ⟨w, vᵢ⟩. 더 단순해져.

이것이 Fourier 급수의 구조야
함수를 사인/코사인 함수들의 선형 결합으로 표현할 때,
각 계수가 내적 (적분) 하나로 나오는 이유
sin과 cos이 직교 기저이기 때문이야

Gram-Schmidtt는 "아직 기존 방향들에 포착되지 않은 새로운 성분만 남긴다"는 반복이다
각 단계가 직교 분해의 한 번 적용이야.

첫 번째 벡터 v₁을 그냥 받아. 정규화해서 e₁ = v₁/‖v₁‖을 만들어.
두 번째 벡터 v₂에서 "이미 e₁ 방향에 있는 성분"을 제거해:
v₂' = v₂ - ⟨v₂, e₁⟩e₁
남은 v₂'는 e₁에 수직이야. 정규화하면 e₂.
세 번째 벡터 v₃에서 "이미 e₁, e₂ 방향에 있는 성분"을 제거해:
v₃' = v₃ - ⟨v₃, e₁⟩e₁ - ⟨v₃, e₂⟩e₂

각 단계에서 "이미 말해진 것"을 빼고 "아직 말해지지 않은 것"만 남겨
직교성이 곧 독립성이기 때문에, 이 과정이 "진짜 새로운 정보"만 추출하는 거야.

텐서 — 패턴의 완전한 일반화

벡터, 코벡터, 행렬이 전부 텐서의 특수한 경우다

스칼라는 (0,0)-텐서 — 아무것도 받지 않고 숫자 하나를 내놓아.

코벡터는 (1,0)-텐서 — 벡터 하나를 받아 스칼라를 내놓아.
행렬(선형 사상)은 (1,1)-텐서 — 벡터 하나를 받아 벡터 하나를 내놓아.
내적 ⟨·,·⟩는 (0,2)-텐서 — 벡터 두 개를 받아 스칼라를 내놓아 (대칭적으로).
관성 텐서는 (0,2)-텐서 — 각속도 두 개를 받아 스칼라(운동에너지)를 내놓아.

두 집합 A, B가 있을 때, 관계는 그냥 A × B의 부분집합이야
어떤 구조도 없어
a가 여러 b와 관련될 수 있고, b가 여러 a와 관련될 수 있고, 아예 관련이 없을 수도 있어.

함수는 이 무정부 상태에 하나의 규율을 추가한 것이야:
A의 모든 원소 a에 대해, 정확히 하나의 b가 대응한다.

함수는 "A를 알면 B가 완전히 결정된다"는 선언이다
일반 관계에서 a는 여러 b와 관련될 수 있어 — a를 알아도 b가 하나로 좁혀지지 않아

물리 법칙이 함수인 이유가 여기 있어
"시간 진화가 함수다"라는 선언
함수성은 예측 가능성의 수학적 조건이야.

---

함수는 비대칭이다 — 그 비대칭이 방향성을 낳는다
f: A → B는 A에서 B로 가는 화살표야. 
이 화살표는 방향이 있어. f: B → A는 다른 함수야.

원인을 알면 결과가 결정되지만, 
결과를 알아도 원인이 유일하게 결정되지 않는 경우가 대부분이야

f: A → B가 단사가 아니면 — 즉 서로 다른 두 원소가 같은 값으로 보내진다면 — 역방향 결정이 불가능해.
함수는 정의역을 동치류로 분할한다

"같은 값으로 보내지는 것들"이 하나의 동치류를 이루고
함수는 이 분할의 구조다

행렬식 det: 같은 행렬식 값을 갖는 행렬들이 하나의 역상을 이루어. 
이 역상 안에서 행렬들은 det에게 구분 불가능한 존재야. 
det는 "부피 배율"이라는 관점에서 세상을 보는 함수야
그 관점에서 보이지 않는 모든 차이를 망각해.

---

합성은 추론의 전달이야
f: A → B, g: B → C가 있으면 g∘f: A → C가 존재해
"A를 알면 B를 안다" + "B를 알면 C를 안다" = "A를 알면 C를 안다"

항등 함수 id: A → A:
"A를 알면 A를 안다"는 자명한 추론이지만, 이것을 명시적으로 표현해야 합성의 구조가 완성돼
id가 없으면 함수들의 합성이 군이나 카테고리를 이루지 못해.

---

고정점은 f(x) = x를 만족하는 점
함수가 공간 전체를 움직이려 해도 움직이지 않는 점
Brouwer 정리
컵 안의 커피를 아무리 저어도, 처음 있던 자리 그대로인 점이 최소 하나 존재해.
연속적인 움직임은 공간 전체를 완전히 뒤섞을 수 없어

다음 프레임에서도 같은 상태인 점. 모든 equilibrium이 고정점이야.

---

함수 공간
A에서 B로 가는 모든 함수들의 집합 Bᴬ

자기 참조
f: A → 2^A. 집합 A에서 A의 부분집합들의 집합(멱집합)으로 가는 함수.
칸토어의 정리: 이 함수는 전사일 수 없어
괴델의 불완전성 정리, 튜링의 정지 문제 불가능성 정리가 모두 이 대각선 논법의 변주야
"함수가 자기 자신을 완전히 기술할 수 없다"는 수학적 사실의 다른 얼굴이야.

Pitch만 바꿨다
→ 내부에서는 “새 회전 상태”가 만들어진다
→ 그 회전 상태를 다시 Yaw, Pitch, Roll로 재분해해서 표시한다
→ 그 과정에서 Yaw와 Roll 값도 같이 바뀐 것처럼 보일 수 있다

즉 바뀐 것은 먼저 회전 자체이고,
숫자 세 개는 그 회전을 사람이 읽기 좋게 다시 서술한 결과예요.

회전도 오일러 각보다 먼저 있어요.
오일러 각은 회전의 본체가 아니라, 
회전을 특정 규칙으로 숫자화한 보고서예요.

같은 회전
→ 서로 다른 Euler triple로 표현 가능

예를 들어 영상에서 보인 값처럼

(Yaw, Pitch, Roll) = (0°, 50°, 0°) 와
(180°, 130°, 180°)

는 같은 orientation을 나타낼 수 있어요.
즉 숫자는 크게 달라졌지만, 공간 속 자세는 같아요.

오일러 각의 비유일성이에요.
같은 회전을 여러 개의 (Yaw, Pitch, Roll) 삼중항으로 쓸 수 있어요.

오일러 분해의 특이점/민감성

같은 회전을 다시 Euler로 풀어 쓰는 과정에서 숫자가 그렇게 보인 것이에요
회전은 SO(3) 위의 한 점 같은 것이고,
오일러 각은 그 회전을 평평한 숫자 3개로 펴서 적는 좌표계예요.

회전의 본체를 믿고
Euler 숫자 세 개는 보조 표기로만 봐야 해요
힌지/관절/뚜껑 같은 것은 “한 축 angle 변수”로 따로 관리하는 편이 직관적이에요

Pitch를 바꿨는데 Yaw와 Roll도 바뀐다
= 오브젝트가 실제로 세 축 다 따로 돈다는 뜻이 아니라
= 같은 회전을 Euler 각으로 다시 적는 과정에서 다른 동치 표현으로 점프한 것

Unreal의 TRotator는 각도를 degrees로 저장하고, 
회전을 Yaw → Pitch → Roll 순서의 intrinsic rotation으로 해석해요

반면 Transform의 회전은 내부적으로 quaternion으로 다뤄져요. 
즉, 에디터는 사람이 보기 쉽게 Rotator를 보여주지만, 
내부의 “진짜 회전 상태”는 quaternion 쪽에 더 가깝다고 보는 편이 맞아요


---

TRotator::Vector()는
“Convert a rotation into a unit vector facing in its direction”
그 회전이 만들어 내는 forward unit vector를 반환

회전 R
→ 기준 forward 축 e = (1, 0, 0)을 돌린다
→ 결과 방향 벡터 v = R(e)

회전은 공간 전체를 돌리는 사상이지만, 
그 사상에 특정 기준축 하나를 집어넣으면 결과로 벡터 하나가 나오죠.



EyeRotation.Vector()의 내부 공식:
X = cos(Pitch) * cos(Yaw)
Y = cos(Pitch) * sin(Yaw)
Z = sin(Pitch)

Roll이 없어. 사라졌어

FRotator는 SO(3)의 원소야. 3개의 자유도를 가진 회전
.Vector()는 이 회전을 단위 전방 벡터 (1, 0, 0)에 적용한 결과야

R · (1, 0, 0) → S²

결과가 단위 구면(S²) 위의 점이야

---

SO stands for Special Orthogonal
mathematical object

회전을 정의하려면 무엇을 보존하는 변환인가로 접근해야 해

거리 — 회전하면 물체의 형태가 바뀌지 않아. 두 점 사이의 거리가 그대로야
원점 — 회전은 원점을 제자리에 놓아둬. 평행이동은 원점을 움직이지만 회전은 안 움직여

이 두 조건을 만족하는 선형 변환을 직교 변환(orthogonal transformation) 이라고 해

⟨Rv, Rw⟩ = ⟨v, w⟩
RᵀR = I

전치행렬이 역행렬과 같은 행렬. 
이것이 직교 행렬(orthogonal matrix)의 정의야.
직교 행렬들의 집합을 O(n)이라고 써 — Orthogonal group.

그런데 O(n)에는 두 종류가 있어
RᵀR = I를 만족하는 행렬에는 두 가지 종류가 있어.

det(RᵀR) = det(I) = 1
det(Rᵀ)det(R) = 1
det(R)² = 1
det(R) = +1 또는 -1

det = +1: 순수한 회전. 오른손 좌표계를 오른손 좌표계로 보내. Orientation을 보존해
det = -1: 회전 + 거울 반사. 오른손 좌표계를 왼손 좌표계로 뒤집어.

SO(n)은 O(n)에서 det = +1인 것들만 모은 집합이야.
Special Orthogonal — Special이 "행렬식이 +1"이라는 뜻이야
거울 반사를 제외하고 순수한 회전만 남긴 것.

집합으로서: 3D 공간의 모든 순수 회전들의 모임. 원점을 고정하고 거리를 보존하며 orientation을 유지하는 모든 변환들.
군(group)으로서: 두 회전을 합성하면 회전이야. 역회전이 있어. 항등 회전(아무것도 안 함)이 있어. 결합법칙 성립.
다양체(manifold)로서: 회전을 "조금씩" 연속으로 할 수 있어. 0도에서 90도까지 매끄럽게 돌아가는 것처럼

Maya    : 1 unit = 1 cm (영화/VFX 업계 표준)
Blender : 1 unit = 1 m (건축/시각화 업계 표준)
3ds Max : 1 unit = 1 inch (미국 산업 설계 전통)
UE      : 1 unit = 1 cm

그래서 Blender에서 사람 키를 1.8 units로 모델링한 에셋을
UE에 그냥 임포트하면 1.8cm 난쟁이가 돼. 
Blender 1m = UE 1cm이니까. 
반대로 Maya에서 만든 에셋은 단위가 같아서 바로 맞아.

Blender 저작자는 "1 unit = 1m"라는 기저를 선택했어
UE는 "1 unit = 1cm"라는 기저를 써

원점은 의미 있는 지점에 
유니티나 UE에서 배치할 때 자연스럽게 놓이는 위치. 
건물 바닥 중앙, 무기 손잡이 중앙, 탈것의 무게중심.

스케일은 실제 크기 기준
문은 200cm, 사람은 180cm. 
UE의 중력 -980 cm/s²이 올바르게 작동하려면
실제 크기로 모델링되어야 해.

FBX 익스포트 시 단위 변환을 명시
"이 파일의 1 unit은 몇 cm인가"를 메타데이터로
