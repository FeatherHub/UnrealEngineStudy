Rendering Equation

나가는 빛 = 표면 자체가 방출한 빛 + 모든 방향에서 들어온 빛이 표면에 반사된 결과

Lo(x, ωo) = Le(x, ωo) + ∫ Ω  fr(x, ωi, ωo)   Li(x, ωi)   (n · ωi)   dωi


n·ωi 표면에 빛이 들어오는 기하학적 각도 효과
n은 위치 x의 surface normal

---

BRDF = Bidirectional Reflectance Distribution Function

fr(x, ωi, ωo)

이미 이 지점에 도달한 빛에 표면이 국소적으로 어떻게 반응하는가?
방향 ωi에서 빛이 들어왔을 때, 방향 ωo로 얼마나 반사되는가

관례, OpenGL 방식


점은 row vector: [x y z 1]
카메라는 -Z 방향을 바라봄
Near 위치: z = -n
Far 위치:  z = -f
NDC 범위: x,y,z ∈ [-1,1] 

---

  ┌ 2n/(r-l)        0               0              0  ┐
  │                                                   │
  │    0         2n/(t-b)           0              0  │
  │                                                   │
  │ (r+l)/(r-l)  (t+b)/(t-b)   -(f+n)/(f-n)       -1  │
  │                                                   │
  └    0             0          -2fn/(f-n)         0  ┘

---

2n/(r-l), 2n/(t-b)
→ Frustum의 가로·세로 크기를 표준 범위에 맞춤

(r+l)/(r-l), (t+b)/(t-b)
→ 비대칭 Frustum의 중심을 NDC 중앙으로 맞춤

-(f+n)/(f-n), -2fn/(f-n)
→ Near/Far depth를 NDC [-1,+1]에 맞춤

-1
→ wc=-z를 만들어 Perspective Divide와 원근 축소를 발생시킴

---

대칭 frustum이라면 훨씬 단순해진다
보통 카메라는 좌우·상하 대칭입니다.

l = -r
b = -t

그러면 중심 보정 항이 사라집니다.

(r+l)/(r-l) = 0
(t+b)/(t-b) = 0

┌ n/r    0         0               0 ┐
│  0    n/t        0               0 │
│  0     0    -(f+n)/(f-n)        -1 │
└  0     0    -2fn/(f-n)           0 ┘

---

n/t = 1 / tan(FOV y/2)

---

Projection Matrix는 네 가지 일을 준비합니다.

view space 절두체를 [−1,1]³ 큐브로 보내는 것

1. Frustum의 폭을 맞춘다
2. 비대칭 중심을 맞춘다
3. Near/Far를 depth 범위에 맞춘다
4. depth를 w에 넣어 원근 나눗셈을 준비한다
