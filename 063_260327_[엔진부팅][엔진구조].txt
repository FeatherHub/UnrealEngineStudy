RegisterComponent() → OnRegister()
    = "이 컴포넌트가 World에 존재함을 World에게 알림"

InitializeComponent()
    = "게임 로직 초기화"
    = 타이머 설정, 다른 컴포넌트 참조 캐싱 등
    = 반드시 RegisterComponent() 이후에 호출됨

운영체제의 로더 — ntdll.dll가
PE(Portable Executable) 헤더를 파싱하고,
이 프로그램이 의존하는 DLL들을 메모리에 매핑하고
각 DLL의 초기화 루틴을 실행한 뒤

마지막으로 CRT(C 런타임)에 제어를 넘긴다.

CRT는
전역 변수들을 0으로 초기화하고
그 다음 전역 객체들의 생성자를 순차적으로 호출

*.gen.cpp 파일들 안에는 다음과 같은 패턴의 전역 객체들이 존재한다.
static TClassCompiledInDefer<AMyActor> AutoInitializeAMyActor(TEXT("AMyActor"), ...);

이 전역 객체의 생성자는 "나중에 AMyActor라는 UClass를 등록해야 한다"는 사실을 
내부 대기열에 적어놓는 것이 전부다
실제 UClass 생성은 아직 아니다

이후 CRT는 WinMain을 호출하고
WinMain은 GuardedMain을 호출한다

GuardedMain의 구조
GEngineLoop
PreInit -> Init -> Tick+ -> Exit

`RHIInit()`
DXGI 팩토리가 생성되고, 
설치된 GPU 어댑터들이 열거되며, 
어댑터 하나가 선택된다

`D3D12CreateDevice()`가 GPU 드라이버와 통신 채널을 수립하고, 
Command Queue(Direct, Compute, Copy), Descriptor Heap들이 생성된다.

`GDynamicRHI`에 이 모든 것을 관장하는 
`FD3D12DynamicRHI` 인스턴스가 배정된다.

이 시점에는 SwapChain이 만들어지지 않는다. 
SwapChain은 OS 창(`HWND`)에 바인딩되어야 하는데, 
창은 아직 만들어지지 않았다.

렌더 스레드(`FRenderingThread`)와 RHI 스레드(`FRHIThread`)가 시작된다
게임의 렌더링 파이프라인이 형성된다

GameThread → [RenderCommand Queue] → RenderThread → [RHICommand Queue] → RHIThread → GPU
