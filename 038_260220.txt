  물류 센터의 모든 박스(Actor)에는 여러 개의 **스티커(Tag Container)**
  단순히 "식품"이라고만 적혀 있지 않고, 식품.냉동.육류 같은 계층 정보

  오직 **스티커(Tag)**만 읽습니다.

Hierarchical Query
  광범위한 체크와 세부적인 체크를 동시에 수행
  포괄적 규칙: "식품.*" 스티커가 붙은 모든 박스는 냉장 창고로
  상세 규칙: "식품.냉동.육류" 스티커가 붙은 박스는 영하 20도 칸으로

Tag System

객체는 속성들의 다발 (Bundle) 이다
속성들의 조합이 객체다

시간성: 속성들이 시간에 따라 이합집산함

정체성은 현재 속성들의 집합
  객체를 “무엇인지”가 아니라 “무엇에 해당하는 명제들이 지금 참인가”로 봅니다

당신은 (지금) 어떤 속성들을 가지고 있습니까? 
  참인 술어 집합이 바뀜
  이에따라 현재 속성 조합에 따라 통과할 수 있는/block되는 경로(Flow)들이 바뀜

여러 속성들이 교차가 현상적 "본질"이 나타나게/발생되게 한다. (Event: 도래, 사건)
  객체는 무한 차원의 교차로이다.

그 Tag를 붙였다가 떼었다가 하는 것은?
  상황

Comparison

Interface vs Component
  Component: 구현 공유 가능
  Interface: 구현 공유 불가능

  Interface: Simple Polymorphism + 계약 정의

Component vs Tag
  Tag: light-weight attribute flags (true or false)
  Component: 무거움 (데이터 + 로직)

  Tag: 데이터+로직을 Encapsulate 할 수 없음

Class vs Tag
  Class: 불변 구조 표현
  Tag: 구조적 보장 결여

Entropy = log₂(가능한 상태의 수)

임의의 문자열 시퀀스 --- Nominal Type
n개의 독립 태그 (2^n 조합) --- Class Hierarchy (N)

Low Entropy → High Entropy
"엔트로피 댐" 건설
- Class: 최소 엔트로피 (강제된 구조)
- Interface: 중간 (계약으로 제약)
- Component: 중간 (조립 규칙)
- Tag: 최대 엔트로피 (자유 상태)

Class 모든 것이 결정된 질서의 감옥
Class가 “너는 영원한 적이다”라고 못 박는 순간,
Tag는 “지금은 적이지만, 상황에 따라 친구가 될 수도 있다”라고 속삭인다.

LV 1 Flat Tag Container 2^N개의 차이
  Red,Low,Collectible,1kg,Sweet

LV 2 Tag Container (Hierarchically structured) (boolean flag) 2^N개의 차이 + 구조화
  Fruit.Apple.Red, Economic.Low, Narrative.Collectible, Physical.Sphere.1kg, Taste.Sweet

LV 3 Tag Container + Quantity (Associated Value) - 무한개의 차이
  Fruit.Apple.Red: {200,30,50}, 
  Economic.cost: {300, 'low'}, 
  Narrative.Collectible: {by: ['good', 'moral']}, 
  Physical.Sphere: {weight: 1kg}, 
  Taste: {Sweet: 100}

  FindAll(HasTag("Fruit.Apple.Red") 
      WHERE ripeness > 150 
      AND quality > 30)

LV 4 Tag Container + Quantity (Associated Value) - 무한개의 차이 + 의미
  Fruit.Apple.Red: {200,30,50} // 0~255, RGB
  Economic.cost: {300, 'low'} // 0~10000, 'low'=0~3333, 'middle'=3334~6666 ... 
  Narrative.Collectible: {by: ['good', 'moral']} // 'good'의 의미...

차이/변화(씨앗, 원인)가 다른 여러 개의 차이/변화(발아, Evolution)를 Drive한다. 

사과객체에 Burning 태그 하나가 "추가되는 순간"

한 객체의 변화를 여러 객체들이 보고 있다.
한 객체를 바꾼다는 것은 그 세계에 Broadcast 한다는 것이다.
여러 객체들이 응답한다. Respond
각 목소리 Voice는 다시 세계에 Broadcast된다.

ripple effect, harmony, 
  Wave 1: 직접 청중
  Wave 2: 2차 청중 (Wave 1의 반응을 듣는다)
  Wave 3: 3차 효과
