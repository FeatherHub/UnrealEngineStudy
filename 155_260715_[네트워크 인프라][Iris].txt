          }
      // 쓴 내용을 FReplicationRecord 에 적재 (ack/재전송용)
    }

Descriptor는 하나의 replication state를 구성하는 member들의 layout

---

`FReplicationProtocol`은 특정 object type을 설명
한 `UObject` type은 여러 replication state로 구성될 수 있다. 

전체 Internal State의 크기와 정렬, 
전체 change mask의 크기를 계산

`Actor_7`과 `Actor_8`은 서로 다른 현재 값을 가지지만 
“어떤 replication state들로 이루어지고 각 state가 어떤 layout을 갖는가”라는 type-level 계획은 같다. 

---

Protocol은 무엇을 어떻게 읽을지 설명하지만 구체적으로 어떤 `UObject`를 읽을지는 모른다. 
특정 instance와 결합하는 존재가 `FReplicationFragment`

reflected property 경로에서는 `FPropertyReplicationFragment`가 이 역할을 한다. 

FPropertyReplicationFragment는 `Owner:UObject*`를 가리키며, 
자신이 담당하는 `FReplicationStateDescriptor`를 참조하고, 
송신 측에서는 `FPropertyReplicationState`를 소유한다.

---

`FPropertyReplicationFragment`를 특정 `UObject`에 배치된 측량사라고 생각할 수 있다. 
Descriptor는 측량 지침서이고, Owner는 실제 측량 대상이며, 
`FPropertyReplicationState`는 측량사가 들고 다니는 현장 기록부

`PollReplicatedState()`가 호출되면 Fragment는 자신의 Owner를 
`FPropertyReplicationState::PollPropertyReplicationState()`에 건넨다. 

`FPropertyReplicationState`는 Descriptor에 기록된 `FProperty`를 이용해 
`UObject`의 실제 property 주소를 찾는다. 
그리고 그 값을 자신의 External State Buffer에 들어 있는 이전 값과 비교한다. 
값이 달라졌다면 해당 change mask bit를 표시하고 새 값을 External State에 복사한다.

`FPropertyReplicationFragment`는 이 작업을 특정 Owner에 결합하고 호출하는 instance adapter

External State는 원본 세계의 언어를 유지한 상태 
`FString`, `TArray`, `FVector`, object reference처럼 Unreal의 native value semantics를 보존
이 상태는 property-aware copy와 비교, RepNotify 적용에 적합

`FNetSerializer::Quantize()`는 External State를 Internal State로 내리고, 
`FNetSerializer::Dequantize()`는 Internal State를 External State로 올린다. 
`Serialize()`와 `Deserialize()`는 이 Internal representation을 byte stream과 왕복시킨다.

---

`FNetSerializer`

특정 value type에 대한 operation table
한 type의 native representation과 canonical representation을 정의하고, 
두 표현의 변환과 byte 표현의 변환을 한곳에 모은다. 

`Quantize`, `Dequantize`, `Serialize`, `Deserialize`가 같은 객체에 있는 이유는 
이 네 연산이 동일한 representation 계약을 공유하기 때문 

한 type의 모든 상태 연산을 한 번의 serializer lookup으로 얻고, 
그 연산들이 동일한 config와 representation invariant를 공유하는 구조를 선택

송신 측에서 `FObjectPoller`가 object를 처리하기로 결정하면 
`FReplicationInstanceOperations`가 해당 `FReplicationInstanceProtocol`을 통해 Fragment들을 찾는다.

`FPropertyReplicationFragment::PollReplicatedState()`는 Owner를 읽어 External State를 갱신한다.
이때 `FPropertyReplicationState`가 `FProperty`를 통해 원본 값과 shadow 값을 비교하고 member change mask를 만든다. 

이어서 Descriptor가 지정한 `FNetSerializer::Quantize()`가 
dirty member의 External representation을 Internal representation으로 변환한다. 

---

수신 측에서 `Deserialize()`가 byte representation을 received Internal State로 복원한다. 
`Dequantize()`가 changed member를 temporary External State로 올린다. 
`FPropertyReplicationFragment::ApplyReplicatedState()`는 이 External State를 
`FPropertyReplicationState`의 형태로 감싸고 `PushPropertyReplicationState()`를 호출한다. 

`PushPropertyReplicationState()`는 Descriptor의 change mask를 보고 적용 대상 member를 고른 다음 
`FProperty`로 Owner 안의 실제 주소를 찾아 값을 쓴다. 
적용이 끝난 뒤 RepNotify를 호출한다. 

송신에서 `FProperty`가 `UObject`를 External State로 읽었고, 
수신에서는 같은 `FProperty`가 External State를 `UObject`에 쓴다.
