3 Lifecycle
  3-1 PreInitializeComponents
    컴포넌트 초기화 전에 액터 차원에서 선행 설정할 기회 (정책/환경)
    “게임플레이 시작”이 아니라 “초기화 시작”
    BeginPlay까지 기다리면 늦거나 꼬일 수 있는 종류의 일
  3-2 InitializeComponents
    액터가 가진 컴포넌트를 돌면서 InitializeComponent()
    bAutoActivate, bWantsInitializeComponent
      컴포넌트는 존재만 한다고 바로 동작하는 게 아니라 
      활성/초기화를 거쳐 동작 상태로 들어간다
  3-3 PostInitializeComponents
    여기서 bActorInitialized = true로 액터를 ‘초기화 완료’ 상태로 확정
    이 단계가 없으면 “컴포넌트 초기화가 끝났는지”를 각 코드가 제각각 추측
    => 엔진이 보장하는 경계
  3-4 BeginPlay
    “월드가 실제로 시뮬레이션/네트워크/틱을 시작할 준비가 끝났는가”
    “액터 개인 이벤트”가 아니라 월드/레벨의 시작 상태와 맞물리는 이벤트
      네트워크/차일드 액터 같은 이유로 지연 가능

4 Network
  Replication: 서버에 있는 액터의 상태 => 클라이언트에 자동으로 동기화
  Role: Autonomous Proxy, Simulated Proxy
  NetMode(NM_DedicatedServer), OnRep_*

5 Integration
  월드 파티셔닝(Data Layers, Runtime Grids)과 통합

6 Editor Support
  에디터에서 배치

1 Reflection / Property System 
  1-1 Object-Oriented Middleware

  1-2 UClass
  1-3 Unreal Header Tool
    Pre-processor 단계의 확장
    컴파일러가 코드를 읽기 전, UHT가 헤더 파일을 분석

  2-2
